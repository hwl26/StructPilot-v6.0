# StructPilot v2.0 项目报告

> 版本：v2.0  
> 项目类型：cryo-EM 数据处理陪跑 Agent / 工作流导航系统  
> 说明：本文档为可直接粘贴到 Word 的交付版报告正文草稿

## 1. 项目背景

StructPilot 是一个面向冷冻电镜（cryo-EM）数据处理流程的智能陪跑系统，目标是帮助初学者或需要标准化流程管理的使用者，在 cryoSPARC / RELION 等软件环境中，按照标准步骤完成从原始数据导入、运动校正、CTF 估计、颗粒挑选、二维分类、三维重构，到结果验证和导出的完整流程。

与传统的“工具脚本”不同，StructPilot 的核心定位不是替用户执行计算，而是作为一个“流程教练”和“知识导航器”，在每个阶段给出清晰的操作指引、参数建议、质控提醒和故障诊断建议，从而降低 cryo-EM 处理门槛，减少新手常见失误，提高流程一致性。

---

## 2. 项目目标

StructPilot v2.0 的设计目标主要包括：

1. **流程陪跑**
   - 将 cryo-EM 处理流程拆解为 14 个检查站，逐站引导用户推进。

2. **知识驱动**
   - 将经验性知识沉淀为结构化知识库，支持流程卡片、故障卡片、规则卡片和模板卡片。

3. **多 Agent 协作**
   - 通过 Navigator、Expert、SOP、Memory 等角色协作，实现不同任务分工。

4. **状态机管理**
   - 使用严格的 checkpoint 状态机跟踪每一步状态，支持 `pending / in_progress / passed / failed / skipped`。

5. **会话持久化**
   - 支持本地会话保存、恢复、历史会话选择和长期记忆。

6. **可交付性与可扩展性**
   - 便于后续扩展 SQLite、向量检索、更多知识库、模型接入和 GUI 优化。

---

## 3. 系统总体架构

StructPilot v2.0 采用“UI + LangGraph 编排 + 多 Agent + 知识库 + 持久化记忆”的分层架构。

### 3.1 层级划分

- **UI 层**
  - 使用 Streamlit 构建交互界面。
  - 提供会话切换、当前节点高亮、checkpoint 状态展示、故障建议、报告导出等功能。

- **编排层**
  - 使用 LangGraph 组织节点和状态流转。
  - 控制 Navigator / Expert / SOP / Fault / Memory 的路由。

- **Agent 层**
  - Navigator：流程推进与 checkpoint 状态管理。
  - Expert：参数解释、规则解释、概念说明。
  - SOP：按阶段展开操作步骤。
  - Memory：会话持久化、恢复与历史索引。

- **知识库层**
  - 保存流程节点、故障知识、决策规则、模板等结构化数据。

- **持久化层**
  - 使用 SQLite 保存会话、消息和 checkpoint 记录。

### 3.2 运行逻辑

用户在 UI 中输入问题后，系统先将输入写入会话状态，再由 LangGraph 路由到适当的 Agent：

- 如果用户询问“怎么做”，会优先由 SOP 提供步骤。
- 如果用户询问“这是什么意思”，由 Expert 解释。
- 如果用户输入“报错 / 失败 / exception”等，系统进入故障诊断分支。
- 如果用户输入“完成 / 通过 / ok”，Navigator 负责推进 checkpoint。
- 最终 Memory 负责保存全部状态。

## 5. 核心模块说明

### 5.1 Navigator Agent

Navigator 是 StructPilot 的核心流程控制器，负责：

- 识别当前 checkpoint
- 标记 checkpoint 状态
- 自动推进下一站
- 识别跳过、完成、故障等意图
- 输出阶段指南和质控标准
- 结合故障知识库做初步诊断

#### checkpoint 状态

系统显式支持以下状态：

- `pending`
- `in_progress`
- `passed`
- `failed`
- `skipped`

这种状态机设计使流程更加清晰，也更适合长流程任务管理。

### 5.2 Expert Agent

Expert Agent 用于回答“这是什么意思”“这个参数该怎么理解”等问题，主要作用包括：

- 参数解释
- 决策规则解释
- 基础概念说明
- 针对具体参数给出建议

### 5.3 SOP Agent

SOP Agent 用于将某个流程节点转成步骤化操作说明，帮助用户“按步骤做”，减少漏步和误操作。

### 5.4 Memory Agent

Memory Agent 当前使用 SQLite 持久化，其职责包括：

- 保存当前会话
- 保存完整消息历史
- 保存 checkpoint 记录
- 支持恢复最近会话
- 支持选择历史会话

## 6. LangGraph 流程设计

StructPilot v2.0 使用 LangGraph 组织工作流节点，使系统的执行路径更加明确。

### 6.1 节点设计

- `navigator`
- `expert`
- `sop`
- `fault`
- `memory`

### 6.2 路由逻辑

- 参数解释类输入 → `expert`
- 步骤类输入 → `sop`
- 故障类输入 → `fault`
- 默认输入 → `navigator`
- 每轮结束 → `memory`

### 6.3 设计收益

- 结构清晰
- 逻辑可维护
- 易于扩展新节点
- 便于后续加入更多条件分支

## 7. 知识库设计说明

### 7.1 知识组织原则

知识库采用结构化 JSON，而不是纯文本堆砌。

建议每条知识具备：

- 唯一 ID
- 标题
- 所属阶段
- 所属软件
- 关键字
- 现象描述
- 原因列表
- 解决建议
- 回溯节点
- 优先级 / 置信度

### 7.2 知识分类

1. **流程节点知识**
   - 例如导入、运动校正、CTF、挑颗粒等。

2. **故障知识**
   - 例如 2D 模糊、颗粒裁切、分辨率偏低等。

3. **规则知识**
   - 例如 box size 的选择、CTF 阈值判断等。

4. **SOP 模板**
   - 用于按阶段输出清晰步骤。

### 7.3 未来扩展建议

未来可以将知识进一步拆分为：

- `flows/`
- `faults/`
- `rules/`
- `sops/`
- `templates/`

并通过检索层与推理层协同工作。

## 8. UI 设计说明

StructPilot v2.0 的 UI 目标是“大气、简约、易操作”。

### 8.1 视觉风格

- 使用浅色背景渐变
- 使用卡片式布局
- 保留充足留白
- 文字层级分明
- 强调功能优先而不是装饰优先

- checkpoint 状态表：展示所有节点状态
- 故障建议区：展示当前故障建议与回溯节点
- 快捷按钮：开始、进度、报告、保存、导出
- 对话区：保留自然语言交互入口

### 8.3 用户体验目标

- 降低学习成本
- 快速定位当前阶段
- 快速恢复历史会话
- 一眼看懂流程状态
- 易于交付和演示

## 10. checkpoint 状态机说明

checkpoint 是 StructPilot 的流程核心。

### 10.1 状态定义

- `pending`：尚未处理
- `in_progress`：正在进行
- `passed`：已通过
- `failed`：未通过或出现问题
- `skipped`：已跳过

### 10.2 状态迁移

- `开始` → 第一个检查站进入 `in_progress`
- `完成 / 通过` → 当前站转为 `passed`，自动进入下一站
- `报错 / 失败` → 当前站转为 `failed`
- `跳过` → 当前站转为 `skipped`，进入下一站

### 10.3 价值

- 让流程有明确进度感
- 让每一步都可回溯
- 方便后续统计、报告和审计

## 12. 使用方式

### 12.1 启动方式

在本地项目目录中运行 Streamlit 程序即可。

### 12.2 典型使用流程

1. 打开系统
2. 选择历史会话或新建会话
3. 点击“开始”或输入“开始”
4. 按照 checkpoint 步骤推进
5. 在遇到问题时输入报错内容
6. 查看故障建议和回溯节点
7. 完成后导出报告

### 12.3 适合的输入方式

- “开始”
- “进度”
- “完成了”
- “有报错”
- “box size 怎么设”
- “这一步是什么意思”
- “跳过这一站”

## 13. 项目成果与价值

StructPilot v2.0 已经从一个简单的交互式助手，逐步演化为一个较完整的 cryo-EM 流程陪跑系统。其主要价值在于：

- 将经验知识结构化
- 将流程状态显式化
- 将交互方式自然化
- 将会话记录持久化
- 将复杂流程变得更易理解和管理

对初学者来说，它能提供“边做边学”的陪跑体验；对实验室流程管理来说，它能提供“标准化流程导航 + 记录保存 + 诊断辅助”的能力。

## 15. 结论

StructPilot v2.0 的核心价值在于把 cryo-EM 数据处理流程从“经验驱动、个人记忆驱动”转变为“结构化知识驱动、状态机驱动、会话可追踪驱动”。

它不是简单的聊天机器人，而是一个面向真实 cryo-EM 工作流的陪跑系统。通过多 Agent 协作、知识库组织、严格状态机和持久化记忆，StructPilot 能够帮助用户更稳地推进流程、更快地定位问题、更清楚地理解每一步的意义。
