# CryoSPARC Workflow 导出与导入完整指南

## 📌 快速说明

**StructPilot 可以导出两种文件**：

| 文件类型 | 用途 | 能否导入 CryoSPARC |
|---------|------|------------------|
| **📄 Markdown 实验记录** | 人类可读的实验报告 | ❌ 不能 |
| **📦 CryoSPARC Workflow JSON** | 机器可读的工作流定义 | ✅ 可以 |

---

## 一、两种文件的区别

### 📄 Markdown 实验记录 (.md)

**内容**：
- 实验目标和背景
- 各步骤的参数设置
- 质检结果和注意事项
- 遇到的问题和解决方案

**用途**：
- 实验室内部归档
- 写论文 Methods 部分的参考
- 团队知识分享
- 向导师/合作者展示流程

**示例**：
```markdown
# StructPilot 实验记录报告

## 基本信息
- 项目名称：TRPV1 蛋白单颗粒重构
- 软件：CryoSPARC v4
- 目标分辨率：<3Å

## 采集参数
- 电压：300kV
- Cs：2.7mm
- 像素大小：0.83Å/pixel
...
```

**❌ 不能导入 CryoSPARC**

---

### 📦 CryoSPARC Workflow JSON

**内容**：
- 每个步骤的 Job Type（import_movies, patch_motion_correction...）
- 数据流连接（哪个 Job 的输出连到哪个 Job 的输入）
- 预填参数（Voltage, Cs, Pixel Size, B-factor...）
- Workflow 元数据（名称、描述）

**用途**：
- **一键创建整个流程**（替代手动逐步建 Job）
- 快速复现相同流程
- 标准化实验室 SOP
- 团队共享 Workflow 模板

**示例**：
```json
{
  "name": "StructPilot_a1b2c3d4",
  "description": "Generated by StructPilot v6.0",
  "nodes": [
    {
      "uid": "node_01",
      "type": "import_movies",
      "params": {
        "voltage_kv": 300,
        "cs_mm": 2.7,
        "psize_A": 0.83
      }
    },
    {
      "uid": "node_02",
      "type": "patch_motion_correction",
      "params": {
        "patch_motion_bfactor": 150
      }
    },
    ...
  ],
  "connections": [
    {
      "from": "node_01",
      "from_slot": "imported_movies",
      "to": "node_02",
      "to_slot": "movies"
    },
    ...
  ]
}
```

**✅ 可以导入 CryoSPARC**

## 二、如何导出 CryoSPARC Workflow JSON

### 方法1：任务完成页（最简单）

1. 完成所有流程步骤
2. StructPilot 自动显示「任务完成」页面
3. 点击「📦 CryoSPARC Workflow (JSON)」按钮下载

### 方法2：高级模式设置（随时可用）

1. 切换到「高级模式」
2. 点击「⚙️ 设置」tab
3. 滚动到「📥 导出 CryoSPARC Workflow」区域
4. 点击「📦 下载 Workflow JSON」

### 方法3：任务完成页中途导出

即使流程未完成，也可以在完成页导出部分 Workflow。

**注意**：
- 必须选择「cryoSPARC」软件（不是 RELION）
- 必须完成入门模式的需求问答（生成推荐流程）
- 导出的 Workflow 包含 StructPilot 推荐的流程和参数

## 三、如何导入 CryoSPARC Workflow

### 前提条件
- CryoSPARC v3.3+ 已安装并运行
- 已创建 Project
- 浏览器能访问 CryoSPARC GUI（通常是 http://localhost:39000）

### 导入步骤

#### 1. 打开 CryoSPARC GUI

访问你的 CryoSPARC Web 界面：
```
http://your-server:39000
```

#### 2. 进入 Project

- 选择或创建一个 Project
- 点击 Project 名称进入

#### 3. 进入 Workflows 面板

- 点击右侧「Workflows」标签
- 如果没有这个标签，说明 CryoSPARC 版本太旧（<v3.3），需要升级

#### 4. 导入 Workflow

- 点击「Import Workflow」按钮（右上角）
- 选择刚下载的 JSON 文件上传
- CryoSPARC 自动解析并显示预览

#### 5. 应用 Workflow

- 预览无误后，点击「Apply」
- CryoSPARC 自动创建所有 Job 并连线
- 完成！现在可以看到完整的流程图

## 四、导入后的操作

### 检查参数

虽然 StructPilot 已经预填了参数，但建议：
1. 逐个 Job 检查参数是否符合你的数据
2. 特别注意：
   - Import Movies：电影文件路径
   - Patch Motion Correction：B-factor
   - Blob Picker：颗粒直径范围

### 运行 Workflow

**方式1：逐步运行（推荐新手）**
- 点击第一个 Job → Queue
- 等待完成后检查结果
- 确认无误后运行下一个

**方式2：批量 Queue（适合熟手）**
- 选中所有 Job（Ctrl+A）
- 点击「Queue Selected」
- CryoSPARC 按依赖关系自动调度

### 调整参数

如果某个 Job 运行结果不理想：
1. 点击 Job → 右侧参数面板
2. 修改参数
3. 点击「Re-run」重新运行

## 五、Workflow 包含的内容

### 标准流程（12步）

| 步骤 | CryoSPARC Job Type | 参数预填 |
|-----|-------------------|----------|
| Import Movies | `import_movies` | Voltage, Cs, Pixel Size |
| Motion Correction | `patch_motion_correction` | B-factor |
| CTF Estimation | `patch_ctf_estimation` | - |
| Particle Picking | `blob_picker` | Particle Diameter |
| Extract | `extract_micrographs` | Box Size |
| 2D Classification | `class_2D` | - |
| Ab-Initio | `ab_initio_reconstruction` | - |
| Hetero Refine | `hetero_refine` | - |
| Homogeneous Refine | `homogeneous_refine` | - |
| CTF Refinement | `ctf_refinement` | - |
| Sharpening | `sharpening` | - |
| Local Resolution | `local_resolution_estimation` | - |

### 数据流连接

自动连接各 Job 的输入输出：
- Import Movies → Motion Correction (movies)
- Motion Correction → CTF Estimation (micrographs)
- CTF Estimation → Particle Picking (micrographs)
- Particle Picking + CTF → Extract (particles + micrographs)
- Extract → 2D Classification (particles)
- 2D → Ab-Initio (particles_selected)
- ...

### 预填参数

从 StructPilot 问答中获取的参数：
- **Voltage**：电压 (kV)
- **Cs**：球差系数 (mm)
- **Pixel Size**：像素大小 (Å/pixel)
- **Particle Diameter**：颗粒直径 (Å)
- **Box Size**：盒子大小 (pixels)
- **B-factor**：运动校正 B 因子
- **Total Dose**：总电子剂量 (e-/Å²)

## 六、常见问题

### Q1：导入时提示「Invalid JSON」

**原因**：JSON 格式错误或文件损坏

**解决**：
1. 重新下载 JSON 文件
2. 用文本编辑器打开，确认是合法 JSON（不是 Markdown）
3. 检查文件大小（正常应该几KB到几十KB）

### Q2：导入成功但没有 Job 显示

**原因**：可能未点击「Apply」

**解决**：
1. 在 Workflows 面板找到刚导入的 Workflow
2. 点击 Workflow 名称
3. 点击「Apply」按钮

### Q3：某些参数没有预填

**原因**：StructPilot 问答中未提供该参数

**解决**：
1. 导入后手动填写缺失参数
2. 下次使用时在 StructPilot 问答中提供完整信息

### Q4：Job Type 不匹配

**原因**：CryoSPARC 版本差异（v3 vs v4）

**解决**：
1. 升级到 CryoSPARC v4+
2. 或手动调整 Workflow（高级用户）

### Q5：能否修改导入的 Workflow？

**可以**：
- 添加/删除 Job
- 修改连接
- 调整参数
- 保存为新 Workflow 模板

### Q6：能否导出 RELION Workflow？

**目前不支持**：
- RELION 没有官方 Workflow JSON 格式
- StructPilot 目前仅支持 CryoSPARC

## 七、高级用法

### 1. 创建标准化 SOP

1. 在 StructPilot 完成一个完整流程
2. 导出 Workflow JSON
3. 在 CryoSPARC 导入并微调参数
4. 保存为实验室标准 Workflow 模板
5. 团队共享此 JSON 文件

### 2. 批量处理多个数据集

1. 导入 Workflow 到第一个 Project
2. 调整参数并验证流程
3. 导出为新的 JSON（带优化参数）
4. 对后续 Project 直接导入优化后的 JSON

### 3. 对比不同参数设置

1. 导入同一个 Workflow 多次
2. 给每个 Workflow 不同的参数（如不同 B-factor）
3. 并行运行
4. 对比结果选择最佳参数

## 八、对比：手动 vs Workflow 导入

| 操作 | 手动创建 | 导入 Workflow |
|------|---------|--------------|
| 创建12个 Job | 逐个点击添加（10-15分钟） | 一键应用（<1分钟） |
| 连接数据流 | 逐个拖拽连线（5-10分钟） | 自动连接（0分钟） |
| 填写参数 | 逐个 Job 填写（10分钟） | 预填好（1-2分钟检查） |
| 出错风险 | 连线错误、参数遗漏 | 极低 |
| 团队协作 | 口头/文档描述 | 共享 JSON 文件 |
| **总耗时** | **25-35分钟** | **<5分钟** |

## 九、技术细节

### Workflow JSON 结构

```json
{
  "name": "Workflow名称",
  "description": "描述",
  "nodes": [
    {
      "uid": "唯一标识符",
      "type": "CryoSPARC Job Type",
      "title": "显示名称",
      "params": {
        "param_key": "param_value",
        ...
      }
    },
    ...
  ],
  "connections": [
    {
      "from": "源节点UID",
      "from_slot": "输出槽名称",
      "to": "目标节点UID",
      "to_slot": "输入槽名称"
    },
    ...
  ],
  "meta": {
    "generated_by": "StructPilot v6.0",
    "timestamp": "2026-07-24T12:00:00",
    "software_version": "cryoSPARC v4"
  }
}
```

### 参数映射

StructPilot → CryoSPARC 参数名映射：

| StructPilot | CryoSPARC Param Key | Job Type |
|-------------|---------------------|----------|
| `voltage` | `voltage_kv` | import_movies |
| `Cs` | `cs_mm` | import_movies |
| `pixel_size` | `psize_A` | import_movies |
| `bfactor` | `patch_motion_bfactor` | patch_motion_correction |
| `particle_diameter` | `diameter_A` | blob_picker |
| `box_size` | `extract_box_size_pix` | extract_micrographs |

## 十、总结

### ✅ CryoSPARC Workflow 导出的优势

1. **省时间**：5分钟完成25分钟的手动操作
2. **减少错误**：自动连线，避免人为失误
3. **可复现**：保存 JSON 即可随时重现流程
4. **易分享**：团队共享标准化 SOP
5. **可追溯**：JSON 文件记录完整参数

### ⚠️ 注意事项

1. **检查参数**：导入后务必检查关键参数
2. **路径问题**：Import Movies 需手动指定文件路径
3. **版本兼容**：确保 CryoSPARC v3.3+
4. **软件选择**：StructPilot 必须选择「cryoSPARC」
5. **完成问答**：必须完成入门模式需求问答
