## 🎯 实施阶段划分

### 🔴 第一阶段：核心功能增强（P0）

**目标**：完善模块一和模块二的核心交互体验

**时间**：3-5天

#### 任务清单

**1. 对话来源标注系统（8小时）**
- [ ] 修改检索结果结构，增加来源元数据
- [ ] 重构 LLM Prompt，要求结构化输出
- [ ] 实现 `components/answer_display.py`（分层展示）
- [ ] 集成到快速问答和聊天界面
- [ ] 测试：提问 "Motion Correction 的 B-factor 怎么设？"，验证是否显示：
  - 基础原理（蓝色框）
  - 课题组经验（金色框，带作者+状态）
  - 相关讨论（灰色折叠）

**2. QC 回溯分析（12小时）**
- [ ] 创建 `components/qc_analysis.py`
- [ ] 实现文本解析器（提取关键指标）
- [ ] 实现图像分析器（OCR + 指标识别）
- [ ] 集成 LLM 诊断模块
- [ ] 课题组经验检索集成
- [ ] 在高级模式增加「📊 QC 回溯」tab
- [ ] 测试：粘贴 Motion Correction 输出，验证诊断结果

**3. 输入验证/XSS防护（3小时）**
- [ ] 创建 `utils/input_validator.py`
- [ ] 所有用户输入（贡献经验、留言板、笔记）增加验证
- [ ] HTML标签过滤（允许基本 Markdown，禁止 <script>）
- [ ] SQL注入防护（虽然用 JSON，但预防未来扩展）
- [ ] 测试：尝试输入 `<script>alert('xss')</script>`，确认被过滤

**交付物**：
- ✅ 用户提问后能看到分层回答（原理+经验+讨论）
- ✅ 用户能粘贴 CryoSPARC 结果并获得 QC 诊断
- ✅ 所有输入已防护 XSS 攻击

### 🟠 第二阶段：体验优化（P1）

**目标**：提升用户操作便利性

**时间**：2-3天

#### 任务清单

**1. 聊天记录→经验（4小时）**
- [ ] 每条 AI 回答下方增加「🏷️ 标记为经验」按钮
- [ ] 实现对话框（标题/分类/步骤/补充说明）
- [ ] 保存到 `lab_experience_kb.json`（待审核状态）
- [ ] 管理员可在审核面板看到
- [ ] 测试：标记一条对话，验证能在管理员审核面板看到

**2. 官方文档链接（3小时）**
- [ ] 创建 `knowledge_base/cryosparc_doc_links.json`
- [ ] 填充常用参数的文档链接（至少 50 个）
- [ ] 在高级模式参数输入框旁显示「📖 查看官方说明」
- [ ] 点击链接在新标签页打开 CryoSPARC 官方文档
- [ ] 测试：查看 Motion Correction 的 bfactor 参数，验证链接可用

**交付物**：
- ✅ 用户能快速将有用的对话标记为经验
- ✅ 高级模式每个参数都有官方文档链接

### 🟡 第三阶段：智能辅助（P2）

**目标**：减少重复劳动，提升数据质量

**时间**：2-3天

#### 任务清单

**1. 经验去重检测（4小时）**
- [ ] 使用 TF-IDF / Sentence Embedding 计算相似度
- [ ] 提交经验时自动检测相似条目
- [ ] 提示用户："已有类似经验，是否仍要提交？"
- [ ] 显示相似条目供参考
- [ ] 测试：提交相似经验，验证去重提示

**2. 智能描述检测（6小时）**
- [ ] 检测标题是否过于简短（<5字）
- [ ] 检测症状描述是否为空
- [ ] 检测解决方案是否完整（是否包含具体数值/操作）
- [ ] 提交时给出改进建议
- [ ] 测试：提交低质量经验，验证提示内容

**交付物**：
- ✅ 提交经验时能检测重复
- ✅ 提交时能检测描述质量并给出建议

## 🛠️ 技术栈选型

| 模块 | 技术方案 | 理由 |
|------|---------|------|
| **来源标注** | 结构化 Prompt + JSON 解析 | LLM 能力已足够，无需额外模型 |
| **QC 分析** | 正则表达式 + LLM 诊断 | 日志格式相对规范，易解析 |
| **图像识别** | PaddleOCR / EasyOCR | 开源免费，识别率高 |
| **去重检测** | Sentence-BERT | 语义相似度准确 |
| **输入验证** | bleach / html.escape | Python 标准库，成熟可靠 |

## 💡 实施建议

### 顺序建议

**如果时间充足（3周）**：
→ 按照 Week 1-2-3 顺序完整实施

**如果时间紧张（1周）**：
→ 只做 P0（对话标注 + QC分析 + 输入验证）

→ 只做 QC 回溯分析（用户最需要）

### 协作建议

如果有团队：
- **前端**：UI 组件（answer_display, qc_analysis）
- **后端**：检索结构、LLM 集成、解析器
- **测试**：功能测试、安全测试

如果独立开发：
- 优先做 QC 分析（价值最高）
- 其次做对话标注（差异化优势）
- 最后做智能辅助（锦上添花）
