# Onboarding 界面重设计说明

**版本**：v2.0  
**设计时间**：2026-07-22  
**设计目标**：提升首次使用体验，降低认知负担，增强专业感

---

## 🎨 设计对比

### 原版问题（v1）

| 问题 | 表现 |
|------|------|
| **视觉层次混乱** | 5个问题全部堆叠展示，信息密度过高 |
| **缺少进度感知** | 用户不知道还要回答多久 |
| **选项说明冗余** | 括号里的补充说明太多（如"~200-400 kDa，常见如 TRPV1、GPCR"） |
| **引导动机不足** | 没说清楚"为什么要回答这些" |
| **按钮不突出** | "生成我的专属流程"按钮埋在底部 |

### 新版改进（v2）

| 维度 | 改进方式 | 用户感知 |
|------|---------|---------|
| **分步展示** | 一次只问1个问题 + "上一步/下一步"按钮 | "不会被一堆问题吓到" |
| **进度可视化** | 顶部进度条 + "第X/5步" | "还有3步就完成了" |
| **卡片化布局** | 每个选项用卡片包裹，带图标和说明 | "看起来很专业" |
| **价值前置** | 欢迎页3个卡片：节省时间/参数预填/课题组经验 | "知道回答这些有什么好处" |
| **渐进式披露** | 回答后才展开下一题 | "一步步引导，不着急" |

## 🎯 设计原则应用

### 1. 渐进式披露（Progressive Disclosure）

**理论**：一次只展示用户当前需要的信息，避免认知过载

**应用**：
- 欢迎页 → 第1问 → 第2问 → ... → 确认页
- 每个问题的补充说明放在卡片内，而非括号里

### 2. 即时反馈（Immediate Feedback）

**理论**：用户操作后立即给出视觉反馈

**应用**：
- 选中选项 → 卡片边框变蓝
- 点击"下一步" → 进度条前进
- 完成配置 → 显示绿色勾号

### 3. 价值驱动设计（Value-Driven Design）

**理论**：先说明价值，再要求付出

**应用**：
- 欢迎页先展示3个价值点
- 每个问题顶部说明"为什么问这个"
- 确认页说明"原因"（而非只给结果）

### 4. 减少选择焦虑（Choice Anxiety Reduction）

**理论**：提供足够信息帮助用户决策

**应用**：
- 每个选项附带"典型案例"和"推荐参数"
- 有默认选中项（最常见场景）
- 提供"不确定"选项（避免强迫选择）

## 🛠️ 技术实现

### 核心改动

| 文件 | 改动 |
|------|------|
| `components/onboarding_v2.py` | 新增分步版问答组件 |
| `modes/beginner.py` | 导入改为 `onboarding_v2` |

### 状态管理

```python
st.session_state.onboarding_step  # 当前步骤（0-6）
st.session_state.user_profile     # 临时存储用户选择
st.session_state.onboarding_completed  # 完成标记
st.session_state.recommended_workflow  # 生成的方案
```

### 导航逻辑

```python
if step == 0:  # 欢迎页
    if st.button("开始配置"):
        st.session_state.onboarding_step = 1
        st.rerun()

elif 1 <= step <= 5:  # 问题页
    # 显示问题 + 选项卡片
    if st.button("下一步"):
        st.session_state.user_profile[key] = selected_value
        st.session_state.onboarding_step += 1
        st.rerun()

elif step == 6:  # 确认页
    # 显示推荐方案
    if st.button("确认并开始"):
        st.session_state.onboarding_completed = True
        st.rerun()
```
