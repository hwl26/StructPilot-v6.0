# 课题组经验贡献功能优化方案

## 🎯 四个优化方向解读

### 1️⃣ 自然语言 + 专业术语理解

**问题**：用户输入时术语不统一
- "motion correction" vs "运动校正" vs "MC"
- "CTF" vs "对比度转移函数" vs "相位对比"

**解决方案**：

**A. 术语自动规范化**
```python
# 提交时自动将别名转为标准术语
"MC" → "Motion Correction (运动校正)"
"CTF" → "CTF Estimation (对比度转移函数估计)"
```

**B. 智能标签提取**
```python
# 从文本中自动识别专业术语，生成标签
输入："Motion Correction 时 local motion too large"
自动标签：["Motion Correction", "local motion", "运动校正"]
```

**C. 输入时实时提示**
```python
# 检测到关键词时显示术语卡片
用户输入 "CTF"
↓
💡 相关术语：CTF Estimation (对比度转移函数估计)
   常见参数：defocus range, astigmatism
```

---

### 2️⃣ 审核写入机制

**问题**：直接写入 JSON，缺乏质量控制

**解决方案**：

**A. 三级状态管理**
```
pending    → 待审核（用户提交后默认状态）
approved   → 已通过（管理员审核通过，可在检索中提权）
rejected   → 已拒绝（不显示）
draft      → 草稿（用户保存但未提交）
```

**B. 管理员审核面板**
- 查看所有 pending 条目
- 一键通过/编辑/拒绝
- 批量操作（通过多条相似经验）

**C. 质量自动检查**
```python
提交时自动验证：
✓ 标题长度 >= 5 字符
✓ 解决方案 >= 20 字
✓ 关联了具体步骤
✓ 至少有 1 个症状关键词
```

---

### 3️⃣ 搭配报错截图

**问题**：只能文字描述，报错界面难以说清楚

**解决方案**：

**A. 截图上传功能**
```python
st.file_uploader(
    "上传报错截图或结果图",
    type=["png", "jpg"],
    accept_multiple_files=True,  # 最多3张
)
```

**B. 截图存储**
```
runtime/experience_images/
  lab_001_error.png
  lab_001_result.png
```

**C. 经验卡片中展示截图**
```
🥇 Motion Correction 报错 local motion too large
├─ 症状：运行到第50帧时报错...
├─ 解决：调整 patch size 从 5x5 到 3x3
└─ 📸 相关截图：
   [报错界面] [调整后参数] [成功运行]
```

---

### 4️⃣ 类似情况经验推送

**问题**：用户遇到问题时，无法主动找到相关经验

**解决方案**：

**A. 质检失败时自动推送**
```python
用户质检失败：CTF 分辨率只有 8Å
↓
🥇 检测到类似经验：
  ├─ "CTF 分辨率低于预期的解决方案"（3 人验证）
  ├─ "CTF Estimation 参数优化经验"（2 人验证）
  └─ "冰层太厚导致 CTF 分辨率差"（1 人验证）
```

**B. 对话中关键词匹配**
```python
用户提问："为什么我的 2D 分类结果很模糊？"
↓
系统检索：
  - 关键词：["2D 分类", "模糊", "结果"]
  - 匹配到：3 条课题组经验
↓
回答中插入：
  "💡 课题组有 3 人遇到过类似问题：
   [查看经验 #lab_012]  [查看经验 #lab_034]"
```

**C. 步骤页面主动展示**
```python
# 在入门模式的操作指令下方
st.info(
    "💡 **课题组提示**：本步骤有 5 条已验证经验，"
    "包括常见报错和参数优化技巧。[查看经验库]"
)
```

## 📊 效果预期

优化前：
- 用户遇到问题 → 自己搜索文档 / 问同学 → 解决后没记录
- 经验沉淀：零散、不可复用

优化后：
- 用户遇到问题 → 系统推送类似经验 → 快速解决 → 一键贡献新方案
- 经验沉淀：结构化、可检索、持续积累

**ROI**：
- 新手上手时间：从 2 天 → 0.5 天
- 重复问题咨询量：减少 60%
- 课题组知识库条目：3 个月从 0 → 50+ 条
