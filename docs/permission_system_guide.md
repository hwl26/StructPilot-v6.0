# StructPilot v6.0 — 成员权限管理完整指南

## 一、权限体系说明

StructPilot 采用 **三级权限** 体系：

| 角色 | 图标 | 权限 | 适用人群 |
|------|------|------|----------|
| **🔑 管理员（admin）** | 红色 | 全部权限 | 课题组 PI / 负责人 |
| **👤 成员（member）** | 蓝色 | 查看 + 贡献 + 评论 + 个人笔记 | 课题组研究人员 |
| **👁️ 访客（guest）** | 灰色 | 仅查看 | 外部访问者 / 未登录用户 |

## 三、首次部署配置

### 1. 默认管理员账号

首次启动 StructPilot 时，系统自动创建默认管理员账号：

```
用户名：admin
密码：admin123
```

**⚠️ 重要**：首次登录后，请立即在「⚙️ 设置 → 👥 成员管理 → 修改密码」中修改默认密码！

### 2. 登录入口

**位置**：左侧边栏顶部
- 未登录显示：`👁️ 访客`
- 点击展开后输入用户名和密码
- 登录成功后显示：`🔑 管理员` 或 `👤 成员`

### 3. 添加第一批成员

管理员登录后：
1. 切换到「高级模式」
2. 点击「⚙️ 设置」tab
3. 滚动到「👥 成员管理」区域
4. 填写新成员信息：
   - 用户名（英文，如 zhangsan）
   - 显示名（中文，如 张三）
   - 初始密码（至少6位）
   - 邮箱（可选）
   - 角色（成员 / 访客）
5. 点击「添加成员」

### 4. 通知成员登录信息

添加成员后，管理员需要告知成员：
```
StructPilot 已为你创建账号：
用户名：zhangsan
初始密码：xxx123
访问地址：http://localhost:8501（或部署后的URL）

首次登录后请修改密码。
```

## 四、日常使用场景

### 场景1：普通成员贡献经验

1. 成员登录后，切换到「高级模式」
2. 点击「💡 贡献经验」tab
3. 填写经验表单（可上传视频/截图）
4. 提交后状态为「待审核」
5. 管理员审核通过后，经验正式入库

### 场景2：管理员审核经验

1. 管理员进入「⚙️ 设置」tab
2. 找到「📋 经验审核」区域
3. 看到所有「待审核」的经验
4. 点击「✅ 通过」→ 改为「已验证」，全体可见
5. 点击「❌ 拒绝」→ 删除此条经验

### 场景3：课题组内部讨论（留言板）

1. 成员登录后进入「⚙️ 设置」tab
2. 滚动到「💬 课题组内部留言板」
3. 点击「➕ 新建帖子」发帖
4. 其他成员可回复、讨论
5. **仅本部署成员可见，不会公开到 GitHub**

### 场景4：个人笔记

1. 成员登录后进入「⚙️ 设置」tab
2. 找到「📝 个人笔记」区域
3. 新建笔记、关联步骤
4. 笔记仅自己可见，不与他人共享

### 场景5：访客查看

1. 不登录，默认「访客」角色
2. 可查看完整流程、参数、知识库
3. 无法贡献、无法查看留言板
4. 适合外部合作者、审稿人

## 五、部署后的访问控制

### 本地部署（默认）

- 访问地址：`http://localhost:8501`
- 仅本机可访问
- 不需要额外配置

### 局域网部署（课题组共享）

1. 启动时 Streamlit 会显示：
   ```
   Network URL: http://192.168.1.100:8501
   ```
2. 课题组成员在同一局域网内访问此地址
3. 所有人共享同一套数据
4. 通过账号权限控制操作

### Streamlit Cloud 部署（公开访问）

**⚠️ 安全提示**：
- Streamlit Cloud 部署后，任何人都能访问 URL
- 未登录用户默认为「访客」，只能查看
- **敏感数据（个人笔记、留言板、待审核经验）不会公开**
- 建议修改默认管理员密码

**部署步骤**：
1. 将代码推送到 GitHub
2. 在 Streamlit Cloud 连接仓库
3. 首次访问后立即登录管理员账号
4. 修改默认密码
5. 添加课题组成员账号

## 六、成员权限变更

### 提升成员为管理员

目前不支持通过界面提升。如需添加第二个管理员：
1. 打开 `runtime/config/users.json`
2. 找到目标用户的 JSON 条目
3. 将 `"role": "member"` 改为 `"role": "admin"`
4. 重启 StructPilot

### 降级成员为访客

管理员在「👥 成员管理」中：
1. 展开目标成员卡片
2. 角色下拉菜单选择「👁️ 访客」
3. 点击「修改角色」

### 删除成员

管理员在「👥 成员管理」中：
1. 展开目标成员卡片
2. 点击「🗑️ 删除」
3. 确认后成员无法登录

**注意**：删除成员不会删除他的个人笔记和贡献的经验

## 七、安全最佳实践

1. **修改默认密码**
   - 首次部署后立即修改 admin 密码
   - 密码至少8位，包含字母+数字

2. **合理分配角色**
   - 仅 PI / 负责人设为管理员
   - 研究人员设为成员
   - 外部访问者不创建账号（默认访客）

3. **定期审查成员**
   - 成员离开课题组后及时删除账号
   - 定期检查「👥 成员管理」列表

4. **备份用户数据**
   - 用户配置存储在 `runtime/config/users.json`
   - 定期备份此文件
   - 迁移部署时复制此文件

5. **公开部署注意**
   - Streamlit Cloud 部署时，URL 任何人可访问
   - 敏感功能已自动权限保护
   - 访客只能查看公开内容

## 八、常见问题

**Q：忘记管理员密码怎么办？**
A：删除 `runtime/config/users.json` 文件，重启后恢复默认账号（admin / admin123）

**Q：能同时有多个管理员吗？**
A：可以，但需手动编辑 `users.json` 文件添加

**Q：访客能看到留言板吗？**
A：不能。留言板仅「成员」和「管理员」可见

**Q：成员之间能看到彼此的笔记吗？**
A：不能。个人笔记仅作者本人可见

**Q：删除成员后他的贡献会消失吗？**
A：不会。贡献的经验保留，但作者显示为原用户名

**Q：如何批量导入成员？**
A：目前仅支持逐个添加。如需批量导入，可编辑 `users.json` 文件

**Q：能对接企业 LDAP / SSO 吗？**
A：当前版本不支持。未来版本可能考虑

## 九、数据存储说明

| 数据类型 | 存储位置 | 可见性 |
|---------|---------|--------|
| 用户账号配置 | `runtime/config/users.json` | 仅管理员 |
| 个人笔记 | `runtime/notes/{username}.json` | 仅作者本人 |
| 课题组留言板 | `runtime/lab_board/posts.json` | 成员+管理员 |
| 经验库（已审核） | `knowledge_base/lab_experience_kb.json` | 全部用户 |
| 经验库（待审核） | 同上，status=pending | 仅管理员可审核 |

**备份建议**：定期备份整个 `runtime/` 目录

## 十、升级迁移

如果你从旧版本（无权限系统）升级到 v6.0：

1. **旧数据兼容**
   - 个人笔记自动迁移（按原用户名关联）
   - 经验库保持原样
   - 留言板需重新创建

2. **首次启动**
   - 自动创建默认管理员账号
   - 原有数据不受影响

3. **成员通知**
   - 告知课题组成员新增了权限系统
   - 提供登录账号
   - 说明访客和成员的区别

## 十一、技术架构

### 权限检查流程

```python
from utils.auth import get_current_user, has_permission

user = get_current_user(st.session_state)
if has_permission(user, "contribute"):
    # 允许贡献经验
else:
    st.warning("需要成员权限")
```

### 密码加密

- 使用 SHA256 单向哈希
- 密码明文不存储
- 不可逆破解

### Session 管理

- 登录状态存储在 `st.session_state.current_user`
- 浏览器关闭后自动退出
- 不使用 Cookie（Streamlit 限制）
