# StructPilot v4 体验修复实施记录

> 日期：2026-07-10
> 对应问题：用户实测反馈「LLM 慢、答非所问、自动滚动差」
> 策略：只改少数文件、每改一处 py_compile 校验，不破坏既有交互路径

## 1. 智能自动滚动（P0）

**根因**：聊天容器内已有自动滚动，但整页（main scroller）每次 rerun 回顶部；旧的 `_chat_auto_scroll` 标志三处写入、零读取，形同虚设。

**改动（main.py）**：
- 新增 `inject_smart_scroll()`（定义于 `render_scroll_shortcuts` 之前）：每次 rerun 后按触发源做【整页级】智能滚动，且**仅当用户主动操作（切换/提问/预设）时触发**，平时不抢滚动。
  - `step_workspace`：切换 step → 整页定位到工作区/对话区顶部（`#sp-work-area` 锚点）。
  - `chat_bottom`：新消息 / 预设提问 / 语音发送 → 整页滚动让最新对话可见（`#chat-bottom` 锚点）。
- `state` 初始化后 `st.session_state.setdefault("_sp_scroll_target", "none")`。
- 在 work 两栏顶部插入锚点 `<span id="sp-work-area">`（原 `st.divider()` 之后）。
- `switch_checkpoint`（2682）、预设问题（3078）、快捷按钮（3188）、语音发送（3680）、主聊天输入（3810）均设置对应 `_sp_scroll_target`。
- 删除 4 处死标志 `st.session_state._chat_auto_scroll = True`（grep 已确认清零）。
- 在 `with tab_chat:` 末尾调用 `inject_smart_scroll()`。

## 2. LLM 响应速度（P0）

**改动 1 — 超时 90s → 30s**：`agents/llm_agent.py:195` `file_config.get("timeout", 90)` → `30`（仍可用 `STRUCTPILOT_LLM_TIMEOUT` 环境变量覆盖）。

**改动 2 — 检索 3 次 → 1 次**：`agents/smart_qa_engine.py` `retrieve()` 原对 `expansion.search_queries[:3]` 串行调用 3 次 `retriever.search`，改为合并为 1 次查询（`top_k=8` 保留多 query 召回），并按 `doc_id` 去重。减少约 3 倍检索网络往返。

**改动 3 — 流式输出（根治“慢”的体感）**：
- `agents/llm_agent.py` 新增 `_openai_compatible_rewrite_stream`（OpenAI-compatible `stream=True`，逐块 yield）与 `rewrite_with_metadata_stream`（入口：仅 OpenAI-compatible 真正流式；其它 provider 退化为一次性 yield 完整文本；**任意异常都回退阻塞式 `rewrite_with_metadata`，保证不丢答案**）。
- `graph/app.py` `handle()` 新增 `stream_sink=None` 参数，存入 `self._stream_sink`（`finally` 清理）；`_polish_reply()` 的 LLM 分支读取 `self._stream_sink`，流式累积 token 并实时回调。
- `main.py` `run_command()` 将 `with st.spinner(...)` 替换为 `st.empty()` 占位符 + `_stream_sink` 闭包，实时渲染 token；`app.handle(..., stream_sink=_stream_sink)`；结束后清空占位符。

> 非 OpenAI-compatible（Anthropic/Gemini）与无 Key 基础模式自动走阻塞/即时路径，行为不变。

## 3. 答非所问（P1）

**改动 1 — RAG 注入阈值 0.35 → 0.5**：`graph/app.py:30` `_RAG_INJECTION_MIN_SCORE = 0.5`，过滤低分噪声片段，避免干扰输出。

**改动 2 — 系统提示加硬约束**：`agents/llm_agent.py` `_system_prompt()` 新增第 8 条：**无权威依据时明说“知识库暂无确切依据”，禁止编造参数/步骤/结论**。直接压制 LLM 自由发挥导致的胡说。

> 路由层（navigator_agent.handle_input）已是顺序+针对性调序（已修复「为什么 pixel size 重要」误判为参数建议）。改为加权打分属较大重构、易回归，本次未做，列为后续可选项。
