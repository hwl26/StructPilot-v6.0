## 🎬 演示脚本（5分钟）

### 第1分钟：Onboarding 引导

**操作**：
1. 展示欢迎页 → "这是定制化的价值"
2. 点击"开始配置"
3. 快速回答5个问题（每个2秒）
4. 到达确认页

**讲解**：
> "传统方式是12步全展示，新人不知道自己需要哪些。我们通过5个问题，自动规划定制流程。比如只做2D分类的用户，系统推荐6步，跳过后6步。"

---

### 第2分钟：参数可编辑

**操作**：
1. 展示推荐参数 → "系统已预填推荐值"
2. 说："假设电镜管理员告诉我像素尺寸是0.826"
3. 修改像素尺寸输入框 → 0.826
4. 指向警告文字 → "⚠️ 此参数错误会导致全流程失败"
5. 点击"恢复推荐值" → 值变回0.86

**讲解**：
> "推荐值是经验估算，但用户收数据后会拿到准确值。我们设计为可编辑，关键参数（像素尺寸/电压）用红色标记，推荐参数可折叠。误改后一键恢复。"

---

### 第3分钟：视图切换

**操作**：
1. 点击"确认并开始" → 进入第1步
2. 输入问题："Box size怎么计算？"
3. 切换视图：
   - 🔍 简化 → 只显示公式
   - 📖 详细 → 公式+参数建议
   - ⚙️ 完整 → 公式+推导+依赖链
4. 指向侧边栏 → "这是全局模式"
5. 指向顶部 → "这是回答详细程度"

**讲解**：
> "我们设计了两层控制：侧边栏控制全局交互方式（入门/教学/高级），顶部控制LLM回答详细程度（简化/详细/完整）。层级清晰，用户可灵活组合。"

---

### 第4-5分钟：对比通用AI

**操作**：
1. 打开预先准备的ChatGPT对话截图
2. 并排对比：
   - ChatGPT："通常box size 256-512"
   - StructPilot："你的Krios 300kV + 膜蛋白150Å → 320px"
3. 展示对比表（提前准备PPT）

**讲解**：
> "通用AI三个关键缺陷：1) 无设备感知，2) 无上下文记忆，3) 无组内经验。我们的对比实验显示，StructPilot在专业准确性、可落地性、上下文感知度三个维度均优于ChatGPT。"

### 问题2：参数输入框不可编辑

**原因**：Streamlit缓存问题

**解决**：
```bash
# 终端执行
streamlit cache clear
# 然后重启
streamlit run main.py
```
