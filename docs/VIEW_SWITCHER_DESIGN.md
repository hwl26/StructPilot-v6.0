# 视图详细程度切换器设计说明

**优化时间**：2026-07-22  
**针对问题**：用户困惑"侧边栏模式切换 vs 顶部按钮"的区别  
**解决方案**：顶部改为**视图详细程度切换器**，控制LLM回答风格和界面信息密度

---

## 🎯 设计目标

### 问题背景

**用户困惑**：
- 侧边栏有"选择模式"（入门/教学/高级）
- 顶部也有"快速/教学/专家模式"按钮
- 两者名称相似，但作用不明确

**设计冲突**：
- 如果两者都是"模式切换"，为什么需要两个入口？
- 点击顶部按钮后，侧边栏的选择会改变吗？

---

## ✅ 新设计：两层控制

### 第一层：侧边栏"交互模式"（全局）

**位置**：侧边栏 → "文互模式"

**作用**：**决定整体交互方式**
- 🔧 入门模式：傻瓜式SOP引导
- 🎓 教学模式：原理讲解+测验
- ⚙️ 高级模式：完整控制权

**影响范围**：所有12步的界面布局

---

### 第二层：顶部"当前视图"（局部）

**位置**：主内容区右上角

**新命名**：
```
当前视图：[🔍 简化] [📖 详细] [⚙️ 完整]
```

**作用**：**调节LLM回答详细程度和界面信息密度**
- 🔍 简化：简洁1-2句话，只给结论
- 📖 详细：完整解释，包含为什么（默认）
- ⚙️ 完整：全部细节+参数+公式

**影响范围**：当前对话+界面展示

---

## 📊 三级视图对比

| 维度 | 🔍 简化 | 📖 详细 | ⚙️ 完整 |
|------|--------|---------|---------|
| **LLM回答长度** | 1-2句话 | 3-5段 | 10+段 |
| **是否解释原理** | ❌ | ✅ | ✅ |
| **参数建议** | 只给推荐值 | 推荐值+范围 | 全参数+公式推导 |
| **故障诊断** | 直接给方案 | 方案+原因 | 方案+原因+预防 |
| **界面折叠** | 默认折叠次要信息 | 平衡展示 | 全部展开 |
| **适用场景** | 快速查找答案 | 日常使用 | 深入研究 |

## 🔧 技术实现

### 状态管理

```python
# main.py 初始化
if "output_detail_level" not in st.session_state:
    st.session_state.output_detail_level = "详细"  # 默认
```

### 视图切换器

```python
def _render_output_mode_toggle(compact: bool = False) -> str:
    view_options = {
        "简化": {
            "icon": "🔍",
            "desc": "简洁回答，只给结论",
            "profile": "concise",
        },
        "详细": {
            "icon": "📖",
            "desc": "完整解释，包含原理",
            "profile": "standard",
        },
        "完整": {
            "icon": "⚙️",
            "desc": "全部细节+参数+公式",
            "profile": "comprehensive",
        },
    }

    current_level = st.session_state.output_detail_level

    if compact:
        selected = st.segmented_control(
            "当前视图",
            options=list(view_options.keys()),
            default=current_level,
            format_func=lambda x: f"{view_options[x]['icon']} {x}",
            key="view_detail_selector",
            label_visibility="collapsed",
            help="调节LLM回答的详细程度和界面信息密度",
        )
    else:
        # 完整模式：三列卡片按钮
        ...

    if selected != current_level:
        st.session_state.output_detail_level = selected
        set_output_mode(view_options[selected]["profile"])
        st.rerun()

    return view_options[current_level]["profile"]
```

### LLM Prompt 调整

**已存在的机制**（`response_profiles.py`）：

```python
def response_profile_instruction(profile: str) -> str:
    if profile == "concise":
        return "简洁回答：1-2句话，只给结论，不解释原理。"
    elif profile == "standard":
        return "完整回答：3-5段，包含原理、参数建议、常见问题。"
    elif profile == "comprehensive":
        return "深度回答：全部细节，参数公式推导，故障预防机制。"
```

**调用位置**（`agents/llm_agent.py`）：

```python
system_prompt = (
    "你是 StructPilot 的 cryo-EM 专家助手。\n"
    + response_profile_instruction(response_profile)  # 注入详细程度要求
)
```

## 📖 使用场景示例

### 场景1：快速查找参数（简化视图）

**用户提问**："2D 分类数应该设多少？"

**🔍 简化视图回答**：
> "50类，基于你的颗粒数（约10万）。"

**特点**：
- 只给结论，不解释
- 1句话，10秒内读完

---

### 场景2：理解原理（详细视图）

**用户提问**："为什么要做 Motion Correction？"

**📖 详细视图回答**：
> "Motion Correction 校正样品在电子束辐照下的漂移。冷冻样品在成像时会因辐照损伤和冰层不均而移动，导致影像模糊。
> 
> **关键参数**：
> - B-factor：控制高频信息权重，漂移大时增大到500
> - Dose weighting：补偿累积辐照损伤
> 
> **判断标准**：校正后的影像边缘清晰，颗粒可辨认。"

**特点**：
- 完整解释原理
- 包含参数建议
- 3-5段，适合日常学习

---

### 场景3：深入研究（完整视图）

**用户提问**："Box size 怎么计算？"

**⚙️ 完整视图回答**：
> "Box size 计算公式：
> 
> **基础公式**：
> ```
> box_size = particle_diameter / pixel_size × 1.8
> ```
> 
> **推导依据**：
> - 1.8倍安全系数：确保颗粒边缘不被裁切
> - Nyquist采样定理：box内至少包含2倍直径范围
> 
> **高效尺寸表**（FFT优化）：
> - 优先选择 2^n × 3^m × 5^k 的尺寸
> - 如计算值314 → 向上取320（2^6 × 5）
> 
> **参数依赖链**：
> ```
> pixel_size (输入)
>     ↓
> particle_diameter (估算)
>     ↓
> box_size (计算)
>     ↓
> Fourier crop, batch size (下游参数)
> ```
> 
> **常见错误**：
> - box过小 → FSC在Nyquist不归零
> - box过大 → 内存占用高，速度慢
> 
> **示例**：
> - 膜蛋白TRPV1（150Å）+ Krios(0.86Å/pix) → 314 → **320**
> - 核糖体（250Å）+ Arctica(1.0Å/pix) → 450 → **480**"

**特点**：
- 公式推导
- 参数依赖可视化
- 常见错误+示例
- 10+段，适合深入研究

## 🎯 与侧边栏模式的配合

### 配合逻辑

| 侧边栏模式 | 建议视图 | 原因 |
|-----------|---------|------|
| 🔧 入门模式 | 📖 详细 | 新人需要完整解释，但不要太复杂 |
| 🎓 教学模式 | ⚙️ 完整 | 学习原理时，信息越详细越好 |
| ⚙️ 高级模式 | 🔍 简化 | 熟练用户只需快速确认，不需啰嗦 |

### 智能推荐（可选扩展）

**切换模式时自动调整视图**：

```python
# 入门模式 → 推荐"详细"视图
if st.session_state.app_mode == "beginner":
    if st.session_state.output_detail_level == "完整":
        st.session_state.output_detail_level = "详细"
        st.info("💡 入门模式推荐使用"详细"视图，已为你自动切换")

# 教学模式 → 推荐"完整"视图
elif st.session_state.app_mode == "teaching":
    if st.session_state.output_detail_level == "简化":
        st.session_state.output_detail_level = "完整"
        st.info("💡 教学模式推荐使用"完整"视图，已为你自动切换")
```

### 如何使用视图切换

**位置**：主界面右上角"当前视图"

**使用场景**：
- **快速查参数** → 切换到🔍简化视图

- 切换视图不影响历史回答
- 只影响后续新提问的回答风格
