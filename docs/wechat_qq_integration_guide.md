## 可行性分析

### ❌ 不可行：个人微信/QQ

| 方式 | 可行性 | 原因 |
|------|--------|------|
| 微信个人号Bot | ❌ | 腾讯不开放API，第三方协议会封号 |
| QQ个人号Bot | ❌ | 腾讯不开放API，需要企业认证 |
| 微信小程序 | ❌ | 需要企业主体，审核周期长 |
| QQ小程序 | ❌ | 需要企业主体，审核周期长 |

**结论**：个人微信/QQ 无法直接接入，必须使用企业版或第三方服务

## 方案1：企业微信机器人（推荐）

### 适用场景
- ✅ 课题组已有企业微信
- ✅ 不需要企业认证（群机器人免费）
- ✅ 配置简单（5分钟）

### 配置步骤

#### 第1步：创建企业微信群机器人

1. 在企业微信中创建群聊（或使用现有群）
2. 点击群设置 → 群机器人 → 添加机器人
3. 机器人名称：`StructPilot Bot`
4. 复制 **Webhook URL**（类似：`https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxxxx`）

#### 第2步：在 StructPilot 中配置

1. 打开 StructPilot → 高级模式 → 设置
2. 找到「📱 企业微信机器人」区域（新增）
3. 粘贴 Webhook URL
4. 点击「测试连接」确认可用
5. 启用机器人

#### 第3步：使用

在企业微信群中发送：
```
#经验 [cp_02] Motion Correction 漂移过大
症状：drift plot 超出范围，很多 micrograph 被丢弃
解决：增大 B-factor 从 150 调到 300
标签：运动校正, 漂移
```

机器人自动回复：
```
✅ 经验已记录
标题：Motion Correction 漂移过大
状态：待管理员审核
作者：张三
```

### 优势
- ✅ 免费，无需企业认证
- ✅ 群成员都能用
- ✅ 支持 @ 机器人交互
- ✅ 可自定义回复消息

### 限制
- ⚠️ 仅支持「发送消息」（Bot 主动推送）
- ⚠️ 不支持「接收消息」（需要公网服务器+回调配置）
- ⚠️ 每个机器人每分钟最多发20条消息

### 单向 vs 双向

**单向（已实现，推荐）**：
- 用户在 StructPilot 网页端触发
- Bot 自动推送消息到企业微信群
- 例如：经验审核通过后，Bot 通知群聊

**双向（需要额外配置）**：
- 用户在企业微信群发消息
- StructPilot 接收并处理
- 需要公网IP或内网穿透（ngrok）
- 配置复杂度：⭐⭐⭐⭐

## 方案2：QQ 群机器人

### 适用场景
- ✅ 课题组使用 QQ 群
- ✅ 有一定技术能力（需要部署 go-cqhttp）

### 配置步骤

#### 第1步：部署 go-cqhttp（第三方QQ协议）

⚠️ **风险提示**：使用第三方协议存在封号风险，建议用小号测试

1. 下载 go-cqhttp：https://github.com/Mrs4s/go-cqhttp/releases
2. 解压并运行：
   ```bash
   ./go-cqhttp
   ```
3. 按提示配置QQ账号（建议用小号）
4. 选择通讯方式：HTTP（端口5700）

#### 第2步：StructPilot 集成

创建 `utils/qq_bot.py`（参考 telegram_bot.py 结构）：
```python
import requests

QQ_BOT_URL = "http://localhost:5700"

def send_group_message(group_id: int, message: str):
    """发送QQ群消息。"""
    requests.post(f"{QQ_BOT_URL}/send_group_msg", json={
        "group_id": group_id,
        "message": message
    })

def listen_qq_messages():
    """监听QQ群消息（需要 Webhook 或轮询）。"""
    # 实现略，参考 go-cqhttp 文档
    pass
```

#### 第3步：使用

在QQ群发送：
```
#经验 CTF 估计结果异常
症状：defocus 都是负数
解决：这是正常的，负数表示过焦
```

Bot 回复：
```
✅ 已记录到经验库（待审核）
```

### 优势
- ✅ 支持双向通信
- ✅ QQ 用户基数大
- ✅ 功能丰富（图片、语音、文件）

### 劣势
- ❌ 需要额外部署 go-cqhttp
- ❌ 使用第三方协议有封号风险
- ❌ 需要一个专门的QQ小号

## 方案3：第三方消息转发服务

### 适用场景
- ✅ 不想自己部署服务器
- ✅ 愿意使用第三方平台

### 推荐服务

#### A. 钉钉群机器人（类似企业微信）
- 免费
- 配置简单
- Webhook 方式
- 参考企业微信方案

#### B. 飞书群机器人
- 免费
- 支持双向通信
- 需要创建飞书应用

#### C. 第三方消息桥接服务

**Zapier / n8n（自动化工具）**：
1. 创建自动化流程
2. 触发器：收到微信/QQ消息（通过企业微信/QQ企业号）
3. 动作：发送HTTP请求到 StructPilot API

**示例流程**：
```
微信消息 → 企业微信 → Zapier → StructPilot Webhook → 保存经验
```

## 已实现的替代方案（推荐）

如果你觉得企业微信/QQ配置太复杂，StructPilot 已经实现了两个更简单的方案：

### 1. Telegram Bot ⭐⭐⭐⭐⭐
- **优势**：5分钟配置，无需服务器，功能完整
- **劣势**：需要科学上网
- **配置**：高级模式 → 设置 → Telegram Bot
- **文档**：`docs/telegram_bot_setup_guide.md`

### 2. 邮件转经验 ⭐⭐⭐⭐
- **优势**：国内可用，所有邮箱支持
- **劣势**：5分钟延迟
- **配置**：高级模式 → 设置 → 邮件转经验
- **文档**：`docs/email_to_experience_guide.md`

### 3. 课题组内部留言板 ⭐⭐⭐⭐
- **优势**：完全本地，私密安全
- **劣势**：需要打开网页
- **位置**：高级模式 → 设置 → 课题组内部留言板
