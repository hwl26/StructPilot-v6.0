# StructPilot v6.0 网站部署方案

> 将 StructPilot 从本地运行部署到公网，供更多人使用

---

## 一、部署方案对比

### 方案 A：Streamlit Community Cloud（最简单）

**优点**：
- ✅ 完全免费
- ✅ 零配置，5 分钟部署
- ✅ 自动 HTTPS
- ✅ 自动更新（关联 GitHub）
- ✅ 官方支持，稳定性好

**缺点**：
- ⚠️ 需要公开 GitHub 仓库（或付费私有）
- ⚠️ 资源限制（1GB RAM, 1 CPU core）
- ⚠️ 访问速度可能较慢（服务器在国外）
- ⚠️ 无法自定义域名（免费版）

**适合场景**：演示、小规模使用、快速验证

---

### 方案 B：云服务器部署（推荐）

**优点**：
- ✅ 完全控制（配置、资源、安全）
- ✅ 可自定义域名
- ✅ 访问速度快（国内服务器）
- ✅ 可扩展（升级配置）
- ✅ 数据隐私（不依赖第三方）

**缺点**：
- ⚠️ 需要购买服务器（约 ¥100-500/月）
- ⚠️ 需要一定运维知识
- ⚠️ 需要配置反向代理、SSL 证书

**适合场景**：生产环境、实验室内部使用、商业用途

**服务商推荐**：
- 阿里云（国内）
- 腾讯云（国内）
- AWS 中国（企业级）
- 华为云（国内）

---

### 方案 C：Docker 容器化部署

**优点**：
- ✅ 环境一致性（开发=生产）
- ✅ 易于迁移和扩展
- ✅ 支持多种部署平台
- ✅ 版本管理方便

**缺点**：
- ⚠️ 需要学习 Docker
- ⚠️ 镜像体积较大

**适合场景**：团队协作、CI/CD 流程、Kubernetes 集群

---

### 方案 D：内网部署（实验室专用）

**优点**：
- ✅ 完全隔离，安全性最高
- ✅ 无需公网带宽
- ✅ 数据不出内网
- ✅ 自定义配置灵活

**缺点**：
- ⚠️ 仅内网可访问
- ⚠️ 需要 VPN 支持远程访问

**适合场景**：科研数据保密、企业内部使用

## 四、部署前准备

### 4.1 代码准备

#### 创建 GitHub 仓库（方案 A 必需，其他可选）

```bash

# 4. 推送到 GitHub
```

### 4.2 环境变量配置

创建 `.env.example`（已有则跳过）：

```ini

STRUCTPILOT_LLM_PROVIDER=openai_compatible
STRUCTPILOT_LLM_API_KEY=your_api_key_here
STRUCTPILOT_LLM_MODEL=Qwen/Qwen3-VL-32B-Instruct

# Embedding 配置

STRUCTPILOT_EMBEDDING_BASE_URL=https://api.siliconflow.cn/v1
STRUCTPILOT_EMBEDDING_API_KEY=your_embedding_api_key_here

# 语音转写配置

STRUCTPILOT_AUDIO_BASE_URL=https://api.siliconflow.cn/v1
STRUCTPILOT_AUDIO_API_KEY=your_audio_api_key_here
```

**⚠️ 重要**：不要把 `.env` 文件提交到 Git！

## 五、安全注意事项

### 5.1 API Key 管理

**❌ 错误做法**：
```python
# 硬编码 API Key（危险！）
api_key = "sk-xxxxxxxxxxxx"
```

**✅ 正确做法**：
```python
# 使用环境变量
import os
api_key = os.getenv("STRUCTPILOT_LLM_API_KEY")
```

### 5.2 敏感数据保护

部署时确保排除：
- ❌ `.env` 文件
- ❌ `runtime/` 目录（包含用户会话数据）
- ❌ SQLite 数据库文件
- ❌ 用户上传的图片/音频

### 5.3 访问控制

对于生产环境，建议添加：
- 🔐 用户登录认证
- 🔑 API 访问限制
- 📊 使用量监控
- 🚦 访问频率限制

## 九、常见问题

### Q1: 部署后访问很慢？

**原因**：
- 服务器带宽不足
- 未启用缓存
- 数据库查询慢

**解决**：
- 升级带宽
- 配置 Nginx 缓存
- 优化数据库查询

### Q2: 如何支持多用户？

**方案 1**：会话隔离（推荐）
- 每个用户独立会话 ID
- 数据库按会话隔离

**方案 2**：用户认证
- 添加登录系统
- 基于用户 ID 隔离数据

### Q3: 如何备份数据？

```bash
# 每天自动备份数据库
0 2 * * * /backup.sh

# backup.sh 内容
#!/bin/bash
timestamp=$(date +%Y%m%d_%H%M%S)
cp runtime/memory/memory.sqlite3 /backups/memory_$timestamp.sqlite3
# 删除 30 天前的备份
find /backups -name "memory_*.sqlite3" -mtime +30 -delete
```
