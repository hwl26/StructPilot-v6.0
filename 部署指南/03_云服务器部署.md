# StructPilot 云服务器部署教程（完整版）

> 生产环境部署指南 - Ubuntu 22.04 + Nginx + Supervisor + SSL

---

## 部署架构

```
Internet
   ↓
Nginx (80/443) → 反向代理 + SSL + 缓存
   ↓
Streamlit (8501) → 仅监听 127.0.0.1
   ↓
SQLite + 运行时文件
```

---

## 第一步：购买服务器

**推荐配置**：
- CPU：2 核（最低）/ 4 核（推荐）
- 内存：4GB（最低）/ 8GB（推荐）
- 系统：Ubuntu 22.04 LTS
- 带宽：5Mbps+
- 地域：华东/华北（根据用户位置）

**价格参考**：¥150-500/月

---

## 第二步：连接服务器

```bash
# SSH 连接
ssh root@你的服务器IP

# 首次登录后，更新系统
apt update && apt upgrade -y
```

## 第三步：安装依赖

```bash
# 一键安装所有必要工具
apt install -y \
    python3.10 \
    python3.10-venv \
    python3-pip \
    git \
    nginx \
    supervisor \
    certbot \
    python3-certbot-nginx
```

## 第四步：创建部署目录

```bash
# 创建应用目录
mkdir -p /opt/structpilot
cd /opt/structpilot

# 方式1：从 Git 克隆
git clone https://github.com/你的仓库/StructPilot.git app

# 方式2：上传压缩包
# 本地打包：tar -czf structpilot.tar.gz StructPilot_v6/
# 上传：scp structpilot.tar.gz root@IP:/opt/structpilot/
# 解压：tar -xzf structpilot.tar.gz && mv StructPilot_v6 app

cd app
```

---

## 第五步：配置 Python 环境

```bash
# 创建虚拟环境
python3 -m venv .venv

# 激活
source .venv/bin/activate

# 安装依赖
pip install --upgrade pip
pip install -r requirements.txt
```

---

## 第六步：配置环境变量

```bash
# 复制模板
cp .env.example .env

# 编辑配置
nano .env
```

填入实际配置：

```ini
STRUCTPILOT_LLM_API_KEY=你的API密钥
STRUCTPILOT_EMBEDDING_API_KEY=你的Embedding密钥
STRUCTPILOT_AUDIO_API_KEY=你的Audio密钥
STRUCTPILOT_RUNTIME_DIR=/opt/structpilot/app/runtime
```

创建运行目录：

```bash
mkdir -p runtime/{memory,cache,uploads}
chmod 750 runtime -R
```

---

## 第七步：配置 Supervisor

创建配置文件：

```bash
cat > /etc/supervisor/conf.d/structpilot.conf << 'EOF'
[program:structpilot]
directory=/opt/structpilot/app
command=/opt/structpilot/app/.venv/bin/streamlit run main.py --server.port 8501 --server.address 127.0.0.1
user=root
autostart=true
autorestart=true
redirect_stderr=true
stdout_logfile=/var/log/structpilot.log
stdout_logfile_maxbytes=50MB
stdout_logfile_backups=5
EOF
```

启动服务：

```bash
# 重载配置
supervisorctl reread
supervisorctl update

# 启动应用
supervisorctl start structpilot

# 查看状态
supervisorctl status structpilot

# 查看日志
tail -f /var/log/structpilot.log
```

---

## 第八步：配置 Nginx

创建配置文件：

```bash
cat > /etc/nginx/sites-available/structpilot << 'EOF'
server {
    listen 80;
    server_name structpilot.yourdomain.com;  # 替换为你的域名

    location / {
        proxy_pass http://127.0.0.1:8501;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 86400;
    }

    client_max_body_size 200M;
}
EOF
```

启用配置：

```bash
# 创建软链接
ln -s /etc/nginx/sites-available/structpilot /etc/nginx/sites-enabled/

# 测试配置
nginx -t

# 重载 Nginx
systemctl reload nginx
```

|-----|---------|--------|

```bash

```

## 常见问题

### Q1: 访问报 502 Bad Gateway

**原因**：Streamlit 没有正常启动

**排查**：
```bash
# 查看 Supervisor 状态
supervisorctl status structpilot

# 查看应用日志
tail -100 /var/log/structpilot.log

# 查看 Nginx 错误日志
tail -100 /var/log/nginx/error.log
```

### Q2: 上传文件失败

**原因**：Nginx 文件大小限制

**解决**：
```bash
# 编辑 Nginx 配置
nano /etc/nginx/sites-available/structpilot

# 添加或修改
client_max_body_size 200M;

# 重载
systemctl reload nginx
```

### Q3: 应用崩溃自动重启

**原因**：内存不足或代码错误

**排查**：
```bash
# 查看系统资源
free -h
top

# 查看详细日志
tail -200 /var/log/structpilot.log
```

## 安全加固

### 防火墙配置

```bash
# 安装 ufw
apt install ufw -y

# 配置规则
ufw allow 22/tcp    # SSH
ufw allow 80/tcp    # HTTP
ufw allow 443/tcp   # HTTPS

# 启用防火墙
ufw enable

# 查看状态
ufw status
```

### 定期备份

```bash
# 创建备份脚本
cat > /opt/backup.sh << 'EOF'
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/opt/backups"
mkdir -p $BACKUP_DIR

# 备份数据库
cp /opt/structpilot/app/runtime/memory/memory.sqlite3 $BACKUP_DIR/memory_$DATE.sqlite3

# 备份配置
cp /opt/structpilot/app/.env $BACKUP_DIR/env_$DATE.txt

# 清理 30 天前的备份
find $BACKUP_DIR -name "*.sqlite3" -mtime +30 -delete

echo "Backup completed: $DATE"
EOF

chmod +x /opt/backup.sh

# 添加定时任务（每天凌晨 2 点）
(crontab -l 2>/dev/null; echo "0 2 * * * /opt/backup.sh") | crontab -
```

## 性能优化

### 1. 启用 Gzip 压缩

编辑 `/etc/nginx/nginx.conf`：

```nginx
gzip on;
gzip_vary on;
gzip_proxied any;
gzip_comp_level 6;
gzip_types text/plain text/css text/xml text/javascript application/json application/javascript application/xml+rss;
```

### 2. 配置缓存

在 Nginx 配置中添加：

```nginx
# 静态文件缓存
location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
    expires 30d;
    add_header Cache-Control "public, immutable";
}
```

### 3. 数据库优化

```bash
# 定期执行 VACUUM（每周一次）
(crontab -l; echo "0 3 * * 0 sqlite3 /opt/structpilot/app/runtime/memory/memory.sqlite3 'VACUUM;'") | crontab -
```
