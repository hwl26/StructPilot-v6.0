### 4. 桌宠互动 ❌

**状态**：渲染成功，但**无法互动**

- [x] 企鹅/小猫 SVG 渲染正常
- [x] 右下角固定位置显示
- [x] CSS 动画定义完整（摇尾巴、脸红、睡觉、爱心）
- [ ] **拖拽功能不工作**
- [ ] **点击无反应**（应该弹出对话气泡）
- [ ] **闲置动画不触发**（应该自动打盹）

**根因**：
- JS 事件监听器在 `window.parent.document.getElementById('spPet')` 找不到元素
- Streamlit iframe 结构导致 DOM 查询失败
- `if (!pet || !bubble) return;` 导致 JS 提前退出

**修复建议**：
1. **方案 A**：改用 Streamlit `components.html()` API 渲染企鹅，确保 JS 在正确的 iframe 内执行
2. **方案 B**：将企鹅 JS 改为 `document.getElementById` 或增加兼容模式（`doc.getElementById || document.getElementById`）
3. **方案 C**：使用 Streamlit 的 `st.session_state` 管理企鹅状态，Python 后端控制显示内容

## 🐛 已知缺陷汇总

### 严重问题 ❌

1. **参数显示异常**
   - 位置：参数 Tab → 关键参数区域
   - 表现：显示 `**—**` 而不是实际数值
   - 影响：用户无法获取参数推荐值
   - 优先级：**高**

2. **企鹅互动失效**
   - 位置：右下角桌宠
   - 表现：无法拖拽、点击无反应
   - 影响：趣味性功能缺失（不影响核心功能）
   - 优先级：**中**

### 次要问题 ⚠️

3. **guide_cards 数据不完整**
   - 影响：部分检查点的 SOP/截图 Tab 内容较少
   - 优先级：**中**

4. **图片路径问题**
   - 部分图片使用相对路径，可能在部署后加载失败
   - 优先级：**低**

5. **参数修改功能缺失**
   - 参数 Tab 只展示，无法修改
   - 优先级：**低**（可能是设计如此）
