# 入门模式优化：需求问答前置

**优化时间**：2026-07-22  
**针对场景**：科研小白首次使用，不了解完整12步流程

---

## 🎯 优化目标

**原问题**：小白用户打开应用 → 看到12步完整流程 → 不知道自己需要哪些步骤 → 认知负担过高

**解决方案**：
1. **前置需求问答**：5个问题（1分钟）收集用户画像
2. **自动规划路径**：根据目标（质检/2D分类/3D重构）动态裁剪流程
3. **参数预填充**：根据样品类型、设备、分辨率目标推荐参数

---

## ✅ 已实现功能

### 1. 需求问答组件（`components/onboarding.py`）

**5个问题**：
- Q1: 研究目标（质检 / 2D分类 / 3D重构）
- Q2: 样品类型（膜蛋白 / 大分子 / 小蛋白 / 不确定）
- Q3: 电镜设备（Krios 300kV / Arctica 200kV / 其他）
- Q4: 目标分辨率（粗筛>10Å / 中等5-10Å / 高分辨<5Å）
- Q5: 是否有师兄师姐带（有 / 独立探索）

**生成输出**：
```python
{
    "steps": ["cp_01", "cp_02", ...],  # 需要的步骤
    "skip_steps": ["cp_07", "cp_08", ...],  # 跳过的步骤
    "params": {  # 预填充参数
        "pixel_size": 0.86,
        "voltage": 300,
        "particle_diameter": 150,
        ...
    },
    "reason": "基于你的目标..."
}
```

### 2. 入门模式集成

- **首次启动检测**：`onboarding_completed == False` → 展示问答对话框
- **跳过步骤处理**：当前步骤在 `skip_steps` 中 → 显示"自动跳过"提示 + "继续下一步"按钮
- **参数预填充**：（框架已就绪，需结合实际参数面板实现）

### 3. 侧边栏增强

- **定制流程提示**：显示"定制流程：6步（跳过6步）目标：2D分类"
- **步骤列表标记**：跳过的步骤显示 `⊗ ~~步骤名~~`（删除线）

## 📊 三种目标的流程裁剪

| 用户目标 | 保留步骤 | 跳过步骤 | 典型用例 |
|---------|---------|---------|---------|
| **初步质检** | Import → Motion Corr → CTF (3步) | cp_04~cp_11 (8步) | 确认数据质量，决定是否继续采集 |
| **2D分类** | Import → Motion Corr → CTF → Blob Picker → Extract → 2D Class (6步) | cp_07~cp_11 (5步) | 筛选好的颗粒类别，发表2D分类图 |
| **3D重构** | 全部12步 | 无 | 完整流程，最终得到三维结构 |

## 🎓 设计理念

### 为什么5个问题

- **认知负担最小化**：5个问题 ≈ 1分钟，不超过用户耐心阈值
- **信息密度最优**：足够生成有效推荐，但不过度询问细节
- **渐进式引导**：先抽象目标 → 再具体参数 → 最后支持方式

### 为什么动态裁剪流程

- **降低完成压力**：看到"6步"比"12步"心理负担小50%
- **聚焦核心目标**：小白不需要理解"为什么有12步"，只需"完成我的目标"
- **提升成就感**：更快完成一个完整闭环，获得正反馈

### 为什么参数预填充

- **减少决策疲劳**：小白最怕的是"这个参数该填什么"
- **基于经验推荐**：预填值来自课题组/社区最佳实践
- **保留调整空间**：预填不是强制，用户仍可手动修改
