# StructPilot v5.1 Changelog

## 版本定位

v5.1 是面向交付和多人使用的可靠性版本。对外显示名统一为 `StructPilot v5.1`，内部历史运行目录可继续保留 v4 命名以兼容已有数据。

## 主要变化

- 统一对外品牌为 `StructPilot v5.1`。
- 增加 `version.py`，由统一常量提供应用显示名和版本号。
- 新增 `verify_v5.py`，作为推荐自检入口。
- 补齐 cp_01 到 cp_12 的内置截图兜底资源，共 22 个文件。
- 回答深度升级为真实生成策略：简洁、教学、专家。
- LLM prompt 新增回答焦点识别：参数、质控、故障、操作、决策、概念、多模态、进度、通用问答。
- 每条新生成的 assistant 消息保存回答深度和回答焦点元数据。
- 多模态处理增加状态反馈、识别摘要、置信度和缓存路径。
- 诊断信息默认收起，减少普通用户的信息负担。
- 知识库管理增加草稿、正式、废弃、来源等级、证据等级、风险等级和命中信息。
- 滚动体验改为更稳定的聊天容器优先策略。
- 新增交付与使用教程。
- 新增发布包构建脚本 `build_release.ps1`。

## 兼容说明

- `verify_v4.py` 保留为兼容入口，但推荐使用 `verify_v5.py`。
- `StructPilot_v4_runtime` 等历史路径名不代表当前软件版本，只用于迁移兼容。
- 历史文档 `CHANGELOG_V4.md` 保留，不作为当前版本说明替换。
