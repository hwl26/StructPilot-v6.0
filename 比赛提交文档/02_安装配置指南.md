# StructPilot v6.0 安装配置指南

## 一、系统要求

### 硬件要求

- **CPU**：双核及以上
- **内存**：建议 4GB 及以上
- **磁盘空间**：至少 2GB 可用空间（含依赖）
- **网络**：首次安装需联网下载依赖；基础模式可离线运行

### 软件要求

- **操作系统**：Windows 10/11（主要测试平台）、Linux、macOS
- **Python**：3.10 或更高版本
- **浏览器**：Chrome、Edge、Firefox 等现代浏览器

## 二、快速安装（Windows 推荐）

### 方式一：一键启动（最简单）

1. **解压软件包**
   ```
   解压 StructPilot_v6.zip 到任意目录
   例如：D:\StructPilot_v6\
   ```

2. **双击启动脚本**
   ```
   双击 start.bat
   ```

3. **等待自动初始化**
   - 首次运行会自动创建虚拟环境（`.venv`）
   - 自动安装所有依赖包
   - 执行健康检查
   - 自动在浏览器打开应用

4. **访问应用**
   ```
   浏览器自动打开：http://127.0.0.1:8501/
   ```

### 方式二：手动启动

如果 `start.bat` 无法运行，可手动执行：

```bash
# 1. 进入软件目录
cd D:\StructPilot_v6

# 2. 创建虚拟环境（仅首次）
python -m venv .venv

# 3. 激活虚拟环境
.venv\Scripts\activate

# 4. 安装依赖（仅首次）
pip install -r requirements.txt

# 5. 启动应用
streamlit run main.py --server.port 8501
```

## 三、配置选项

### 基础模式（无需配置）

直接启动即可使用，功能包括：
- ✅ 流程引导和状态管理
- ✅ 本地规则库匹配
- ✅ 内置 SOP 和参数建议
- ✅ 截图兜底展示
- ✅ 会话记录和报告导出

### AI 增强模式（可选）

如需启用 AI 功能（语义检索、图像理解、语音转写），需配置 API：

1. **复制配置模板**
   ```bash
   复制 .env.example 为 .env
   ```

2. **编辑 .env 文件**
   ```ini
   # ============= LLM 配置 =============
   # 服务商：openai_compatible / anthropic / gemini / none
   STRUCTPILOT_LLM_PROVIDER=openai_compatible
   
   # API Key（必填）
   STRUCTPILOT_LLM_API_KEY=your_api_key_here
   
   # 模型名称
   STRUCTPILOT_LLM_MODEL=Qwen/Qwen3-VL-32B-Instruct
   
   # API Base URL
   STRUCTPILOT_LLM_BASE_URL=https://api.siliconflow.cn/v1
   
   # 请求超时（秒）
   STRUCTPILOT_LLM_TIMEOUT=30
   
   # ============= Embedding 配置（用于 RAG 检索）===========
   STRUCTPILOT_EMBEDDING_MODEL=BAAI/bge-m3
   STRUCTPILOT_EMBEDDING_BASE_URL=https://api.siliconflow.cn/v1
   STRUCTPILOT_EMBEDDING_API_KEY=your_embedding_api_key_here
   
   # ============= 语音转写配置（用于语音输入）===========
   STRUCTPILOT_AUDIO_MODEL=FunAudioLLM/SenseVoiceSmall
   STRUCTPILOT_AUDIO_BASE_URL=https://api.siliconflow.cn/v1
   STRUCTPILOT_AUDIO_API_KEY=your_audio_api_key_here
   ```

3. **重启应用**
   - 关闭当前运行的应用（Ctrl+C）
   - 重新运行 `start.bat` 或 `streamlit run main.py`

### 高级配置（可选）

在 `.env` 中可进一步配置：

```ini
# 截图目录配置（外部截图库路径）
# 默认使用内置的 assets/guides/ 截图
STRUCTPILOT_SCREENSHOTS_DIR=D:/custom_screenshots/

# 运行数据目录（会话记忆/上传文件/缓存）
# 默认为 runtime/ 子目录
STRUCTPILOT_RUNTIME_DIR=D:/structpilot_data/
```

## 五、常见问题

### Q1: 启动时提示"端口被占用"

**解决方法**：
```bash
# 方式1：更换端口启动
streamlit run main.py --server.port 8517

# 方式2：查找并关闭占用8501端口的进程
netstat -ano | findstr :8501
taskkill /F /PID <进程ID>
```

### Q2: 虚拟环境创建失败

**可能原因**：
- Python 版本低于 3.10
- pip 未正确安装
- 网络连接问题

**解决方法**：
```bash
# 检查 Python 版本
python --version

# 更新 pip
python -m pip install --upgrade pip

# 使用国内镜像源
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### Q3: 依赖安装超时

**解决方法**：
```bash
# 使用国内镜像源
pip install -r requirements.txt -i https://mirrors.aliyun.com/pypi/simple/

# 或者清华源
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### Q4: 浏览器无法访问应用

**检查项**：
1. 确认终端显示 "You can now view your Streamlit app in your browser."
2. 检查防火墙是否拦截 8501 端口
3. 尝试更换浏览器
4. 确认 URL 是 `http://127.0.0.1:8501/` 而非 `https://`

### Q5: AI 功能不可用

**原因**：未配置或配置错误

**解决方法**：
1. 检查 `.env` 文件是否存在且配置正确
2. 验证 API Key 是否有效
3. 测试网络能否访问 API Base URL
4. 查看应用启动日志中的错误提示

### Q6: 上传图片后无响应

**可能原因**：
- 图片格式不支持（仅支持 JPG/PNG）
- 图片过大（建议小于 10MB）
- 未配置 Vision API

**解决方法**：
1. 压缩图片后重试
2. 检查"诊断信息"中的错误提示
3. 启用 AI 增强模式并配置 Vision API

## 六、离线部署

### 场景：无法联网的实验室环境

1. **在联网机器上准备依赖包**
   ```bash
   # 下载所有依赖到本地目录
   pip download -r requirements.txt -d ./packages
   ```

2. **打包传输**
   ```
   将软件目录和 packages 目录一起打包
   传输到目标机器
   ```

3. **离线安装**
   ```bash
   # 在目标机器上安装
   pip install --no-index --find-links=./packages -r requirements.txt
   ```

## 七、卸载

### 完全卸载

```bash
# 1. 停止运行的应用（Ctrl+C）

# 2. 删除虚拟环境
rmdir /s /q .venv

# 3. 删除运行数据（可选，会删除所有会话记录）
rmdir /s /q runtime

# 4. 删除软件目录
rmdir /s /q D:\StructPilot_v6
```

### 保留数据卸载

如需保留会话记录，仅删除软件目录和虚拟环境，保留 `runtime/` 目录。下次重新安装后可继续使用历史会话。
