# StructPilot v6.0 使用手册

## 一、界面概览

### 主界面布局

```
┌─────────────────────────────────────────────────────────┐
│  StructPilot v6.0                        [设置] [帮助]  │ ← 顶部导航
├─────────────┬───────────────────────────────────────────┤
│             │  [对话] [报告] [知识库]  ← 主功能标签页    │
│  侧边栏      │                                           │
│             │  ┌─────────────────────────────────────┐  │
│ • 软件选择  │  │                                     │  │
│ • 检查点    │  │        聊天对话区                    │  │
│ • 回答深度  │  │        (滚动显示)                    │  │
│ • 会话管理  │  │                                     │  │
│             │  └─────────────────────────────────────┘  │
│             │  [输入框]  [发送] [📷] [🎤]  ← 交互区     │
└─────────────┴───────────────────────────────────────────┘
```

### 侧边栏功能

1. **软件体系选择**
   - cryoSPARC（默认）
   - RELION
   
2. **当前检查点**
   - 显示 12 个标准检查点
   - 绿色：已完成
   - 黄色：进行中
   - 红色：失败
   - 灰色：待开始

3. **回答深度**
   - 简洁：快速决策
   - 教学：详细指导
   - 专家：深度分析

4. **会话管理**
   - 新建会话
   - 切换历史会话
   - 重命名会话
   - 删除会话

## 二、基本使用流程

### 2.1 启动新项目

1. **打开应用**
   ```
   浏览器访问 http://127.0.0.1:8501/
   ```

2. **选择软件体系**
   ```
   侧边栏 → 选择 cryoSPARC 或 RELION
   ```

3. **选择回答深度**
   ```
   简洁：快速操作用户
   教学：学习阶段用户（推荐新手）
   专家：科研决策用户
   ```

4. **开始第一步**
   ```
   输入："开始 cp_01"
   或点击预设问题："如何导入数据？"
   ```

### 2.2 流程推进

#### 标准检查点流程

| 检查点 | 名称 | 主要内容 |
|--------|------|----------|
| cp_01 | 数据导入 | Movies 导入、元数据设置 |
| cp_02 | 运动校正 | Patch Motion Correction |
| cp_03 | CTF 估计 | CTF Estimation |
| cp_04 | 颗粒挑选 | Blob/Template/Topaz Picker |
| cp_05 | 颗粒提取 | Extract Particles |
| cp_06 | 2D 分类 | 2D Classification |
| cp_07 | 2D 筛选 | Select 2D Classes |
| cp_08 | 初始模型 | Ab-Initio Reconstruction |
| cp_09 | 异质重构 | Heterogeneous Refinement |
| cp_10 | 同质重构 | Homogeneous Refinement |
| cp_11 | 非均匀优化 | Non-Uniform Refinement |
| cp_12 | 模型验证 | FSC、局部分辨率评估 |

#### 状态汇报关键词

- **开始某步骤**
  ```
  "开始 cp_02"
  "开始运动校正"
  "开始 motion correction"
  ```

- **完成某步骤**
  ```
  "完成 cp_02"
  "cp_02 做完了"
  "motion correction 完成"
  ```

- **报告失败**
  ```
  "cp_02 报错了"
  "motion correction 失败"
  "出现错误：[错误信息]"
  ```

- **跳过某步骤**
  ```
  "跳过 cp_03"
  "skip ctf estimation"
  ```

```

系统回答内容：
```

#### 操作类问题

## 三、多模态输入

### 3.1 图片上传

#### 适用场景

- CTF 诊断图
- 2D classification 结果
- 3D refinement 密度图
- FSC 曲线截图
- 取向分布图
- 软件报错截图

#### 使用方法

**方式 1：点击上传**
```
1. 点击输入框右侧的 📷 图标
2. 选择本地图片文件（JPG/PNG）
3. 等待上传完成
4. 输入问题："这张 CTF 正常吗？"
5. 点击发送
```

**方式 2：粘贴上传**
```
1. 在其他软件中复制截图（Ctrl+C）
2. 点击输入框
3. 粘贴（Ctrl+V）
4. 图片自动上传
5. 输入问题并发送
```

#### 系统反馈

上传后会显示：
- ✅ 原图缩略图
- 🔍 图像识别摘要（AI 模式）
- 📊 置信度评分
- ⚠️ 关键风险提示

### 3.2 语音输入

#### 适用场景

- 实验室现场快速提问
- 双手忙碌时口述问题
- 长段问题描述

#### 使用方法

```
1. 点击输入框右侧的 🎤 图标
2. 录制或上传音频文件（WAV/MP3/M4A）
3. 等待语音转写完成
4. 确认转写文本
5. 系统自动处理问题
```

#### 注意事项

- 语音需清晰，避免环境噪音
- 专业术语建议中英文对照（如"CTF 估计"）
- 转写结果可手动修改后再发送

### 3.3 纯文本输入

最常用的输入方式，适合：
- 结构化参数问题
- 明确的操作指令
- 状态汇报

## 四、高级功能

### 4.1 会话管理

#### 创建新会话

```
侧边栏 → 会话管理 → [+ 新建会话]
```

新会话会：
- 重置检查点进度
- 清空对话历史
- 保留软件体系选择

#### 切换会话

```
侧边栏 → 会话管理 → 选择历史会话
```

可同时维护多个项目，每个会话独立记录：
- 流程进度
- 对话历史
- 参数记录
- 质控结果

#### 会话命名

```
侧边栏 → 会话管理 → [✏️ 重命名]
```

建议命名规范：
```
项目名_日期_备注
例如：ApoA1_20260718_run1
```

### 4.2 报告导出

#### 查看报告

```
切换到"报告"标签页
```

报告包含：
- **流程总览**：12 个检查点的完成状态
- **参数汇总**：使用过的关键参数
- **质控记录**：每步的 QC 结果
- **完整对话**：时间线式对话记录

#### 导出报告

```
报告页面 → 点击 [导出 Markdown]
```

导出文件可用于：
- 课题组内部交流
- 实验记录归档
- 问题复现和排查

### 4.3 知识库管理

#### 查看知识库

```
切换到"知识库"标签页
```

显示：
- 正式知识（已审核）
- 草稿知识（待审核）
- 废弃知识（已过时）

#### 沉淀经验

在对话中遇到有价值的回答时：

```
1. 点击该回答下的 [沉淀经验] 按钮
2. 编辑知识标题和内容
3. 选择分类（参数/质控/故障/概念）
4. 设置来源等级（官方文档/实验室经验/用户反馈）
5. 保存为草稿
6. 管理员审核后转为正式知识
```

#### 知识评级

| 等级 | 含义 | 用途 |
|------|------|------|
| 正式 | 已审核确认 | 优先用于回答 |
| 草稿 | 待审核 | 仅供参考 |
| 废弃 | 已过时或错误 | 不再使用 |

### 4.4 诊断信息

对于技术用户或排查问题时：

```
展开对话中的"诊断信息"区域
```

可查看：
- 问题理解结果
- 关键词提取
- 知识库检索命中
- RAG 相似度得分
- LLM 调用参数
- 处理耗时

## 五、最佳实践

### 5.1 新手上手流程

**第 1 周：熟悉基本流程**
```
1. 选择"教学"模式
2. 按 cp_01 → cp_12 顺序逐步推进
3. 主动汇报每步"开始"和"完成"
4. 遇到不懂的概念立即提问
5. 每完成一步查看报告页面
```

**第 2 周：深入参数调优**
```
1. 尝试"专家"模式
2. 对比不同参数的效果
3. 上传实际处理的截图
4. 学习质控标准
5. 沉淀有价值的经验
```

**第 3 周：独立处理**
```
1. 切换到"简洁"模式快速决策
2. 仅在卡点处咨询
3. 主动维护知识库
4. 帮助新人解答问题
```

### 5.2 团队协作模式

**知识共建**
```
1. 每人负责 2-3 个检查点的知识维护
2. 遇到新问题及时"沉淀经验"
3. 定期（每周）审核草稿知识
4. 更新过时的参数建议
```

**案例库积累**
```
1. 成功案例：导出报告 + 关键参数
2. 失败案例：记录问题 + 解决路径
3. 边界案例：特殊样本的处理策略
```

### 5.3 提问技巧

**❌ 不推荐的提问**
```
"帮我处理数据"  （过于宽泛）
"怎么办？"  （缺少上下文）
"为什么不行？"  （无具体信息）
```

**✅ 推荐的提问**
```
"cryoSPARC 中 patch motion correction 的 bfactor 参数，
对于 300kV、K3 相机、1.06Å/pixel 的数据，推荐值是多少？"

"我的 2D classification 结果中（上传截图），
哪些 class 应该保留用于下一步？为什么？"

"运行 ab-initio 时报错：cuda out of memory，
我的显卡是 RTX 3090 24GB，粒子数 50 万，box size 256，
应该如何调整参数？"
```

### 5.4 质控检查清单

每完成一个检查点，确认：

**cp_01 数据导入**
- [ ] Movies 数量和总大小
- [ ] Pixel size 正确
- [ ] 电压和球差设置正确

**cp_02 运动校正**
- [ ] 漂移量在合理范围（< 50Å）
- [ ] 未出现大量失败的 movies
- [ ] B-factor 合理（100-500）

**cp_03 CTF 估计**
- [ ] Thon rings 清晰
- [ ] Defocus 范围正常（0.8-3.0 μm）
- [ ] Fit 质量良好（< 5Å）

**cp_06 2D 分类**
- [ ] 出现高分辨率特征
- [ ] 不同视角分布合理
- [ ] 无明显杂质或冰污染

**cp_12 最终模型**
- [ ] FSC 达到目标分辨率
- [ ] 局部分辨率均匀
- [ ] 无明显伪影

## 六、故障排查

### 6.1 应用层问题

**问题：回答一直在"生成中"**

原因：API 超时或网络问题

解决：
1. 检查网络连接
2. 查看终端错误日志
3. 增大 `.env` 中的 `STRUCTPILOT_LLM_TIMEOUT`
4. 切换到基础模式（不依赖 API）

**问题：上传图片后无法识别**

原因：未配置 Vision API

解决：
1. 确认 `.env` 中配置了支持 Vision 的模型
2. 检查 API Key 权限
3. 或使用基础模式（会显示截图但无自动识别）

**问题：会话切换后进度丢失**

原因：数据库写入失败

解决：
1. 检查 `runtime/` 目录权限
2. 确认磁盘空间充足
3. 查看 SQLite 数据库是否损坏

### 6.2 流程层问题

**问题：系统无法识别完成状态**

关键词可能不匹配，尝试：
```
"cp_02 完成"
"完成 motion correction"
"motion correction done"
```

**问题：建议的参数不适合我的数据**

说明：
- 系统给的是通用建议
- 需根据实际情况（粒径/电压/探测器）调整
- 可追问："对于 XX 条件，如何调整？"

### 6.3 知识库问题

**问题：检索不到相关知识**

原因：
1. 知识库内容不足
2. 问题表述与知识条目差异大
3. 未启用 Embedding 检索

解决：
1. 换个问法重新提问
2. 补充更多上下文
3. 配置 Embedding API 启用语义检索
4. 主动补充知识库内容
