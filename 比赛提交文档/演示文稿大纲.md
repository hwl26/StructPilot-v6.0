## 幻灯片 2：问题背景

### 冷冻电镜数据处理的三大挑战

1. **流程复杂**
   - 从原始数据到 3D 结构需要 12+ 个步骤
   - 每步都有大量参数需要设置
   - 新手容易迷失方向

2. **参数难调**
   - 参数取值依赖经验
   - 错误设置导致结果失败
   - 试错成本高（几小时到几天）

3. **知识分散**
   - 文档散落在官网/论坛/论文
   - 实验室经验难以传承
   - 故障排查依赖老师傅

## 幻灯片 3：我们的方案

### StructPilot = 陪跑教练

**核心理念**：不替代专业软件，而是提供全程引导

- 🗺️ **流程导航**：12 个检查点，逐步推进
- 💡 **智能建议**：参数推荐、质控判断、故障诊断
- 📚 **知识库**：实验室经验积累和传承
- 🤖 **AI 增强**：多模态理解（文本/图片/语音）

### 3. 多模态交互

## 幻灯片 5：技术架构

### 五层架构设计

```
表现层 ─────────→ Streamlit UI（对话/报告/知识库）
   ↓
编排层 ─────────→ LangGraph 状态机
   ↓
智能体层 ────────→ Navigator / LLM / Memory
   ↓
知识层 ─────────→ 规则库 / RAG 检索 / 术语库
   ↓
持久层 ─────────→ SQLite（会话/消息/检查点）
```

### 核心设计原则

1. **单一权威应答**：避免重复回复
2. **全链路优雅降级**：任一环节缺失不阻断
3. **代码知识解耦**：知识更新无需改代码
4. **状态可追溯**：完整持久化和回溯

## 幻灯片 7：实际效果演示

### 场景 1：参数咨询

**用户提问**：
```
cryoSPARC patch motion correction 的 bfactor 参数，
对于 300kV K3 相机数据，推荐值是多少？
```

**系统回答**（教学模式）：
- 参数定义：运动校正的 B-factor（平滑因子）
- 推荐值：150-300
- 依据：相机类型、电压、像素大小
- 设置位置：cryoSPARC Job Builder → Patch Motion Correction
- 注意事项：太大会过度平滑，太小会引入噪声

## 幻灯片 9：技术亮点

### 1. 工程实现

- ✅ **纯 Python**：无 C++ 依赖，跨平台兼容
- ✅ **一键启动**：双击 bat 自动初始化
- ✅ **依赖简单**：核心依赖仅 9 个包
- ✅ **部署友好**：虚拟环境隔离，离线可部署

### 2. 性能优化

- **Embedding 缓存**：按内容 hash 复用向量
- **图片压缩**：自动压缩到 1024px
- **语音缓存**：相同音频不重复转写
- **会话管理**：SQLite 索引优化查询

### 3. 安全设计

- **本地运行**：数据不出站
- **敏感信息保护**：API Key 不入库
- **输入校验**：防止注入攻击
- **错误隔离**：单个查询失败不影响整体
