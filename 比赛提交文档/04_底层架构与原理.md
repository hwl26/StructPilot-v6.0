# StructPilot v6.0 底层架构与原理

## 一、整体架构设计

### 1.1 分层架构

StructPilot 采用五层架构设计，实现代码能力与知识内容的完全解耦：

```
┌──────────────────────────────────────────────────────────┐
│  表现层 (Presentation Layer)                              │
│  • Streamlit UI (main.py)                                │
│  • 三个主标签页：对话 / 报告 / 知识库                        │
│  • 侧边栏：软件选择 / 检查点导航 / 会话管理                   │
│  • 多模态输入：文本 / 图片 / 语音                           │
├──────────────────────────────────────────────────────────┤
│  编排层 (Orchestration Layer)                             │
│  • LangGraph 状态机 (graph/app.py)                        │
│  • StructPilotApp：单线图编排                             │
│  • 挂载点 _polish_reply：RAG / Vision / LLM 改写           │
│  • 路由决策：concept / expert / memory                    │
├──────────────────────────────────────────────────────────┤
│  智能体层 (Agent Layer)                                   │
│  • NavigatorAgent：状态机流转、关键词路由                   │
│  • LLMAgent：多模态处理、embedding、改写                   │
│  • MemoryAgent：SQLite 持久化                             │
│  • SmartQAEngine：问题理解、扩展、推理、组合                │
│  • ExpertAgent / SOPAgent：备用智能体                     │
├──────────────────────────────────────────────────────────┤
│  知识层 (Knowledge Layer)                                 │
│  • 知识库：检查点定义 / 故障库 / 专家规则 / SOP / QC 标准   │
│  • RAG 检索器：numpy 余弦相似度 + 缓存 + 阈值过滤           │
│  • 术语库：概念 / 缩写 / 文件格式定义                        │
│  • 文档导入器：知识文档读写 / 索引管理 / 冲突检测            │
├──────────────────────────────────────────────────────────┤
│  持久层 (Persistence Layer)                               │
│  • 运行时状态：PipelineState / Message / CheckpointRecord │
│  • SQLite 数据库：4 张表（sessions / messages / 图像 / 检查点）│
│  • 缓存：Embedding 向量 / 图片缩略图 / 语音转写结果          │
└──────────────────────────────────────────────────────────┘
```

### 1.2 核心设计原则

**原则 1：单一权威应答者**
- NavigatorAgent 是唯一追加助手回复的节点
- 避免多智能体并发产生重复回复
- 所有增强能力（RAG/Vision/LLM）挂载在单一管线上

**原则 2：全链路优雅降级**
- 未配 Embedding → RAG 检索返回空，对话照常
- 未配 Vision → 忽略图片，走纯文本
- 未配 LLM → 退回纯规则模式
- 任何环节缺失都不报错，系统始终可用

**原则 3：代码与知识解耦**
- 空知识库能跑通流程（基于内置规则）
- 导入实验室 SOP 后能力自动增强
- 知识更新无需改代码

**原则 4：状态可追溯**
- 完整持久化会话、消息、检查点记录
- 支持会话恢复和进度回溯
- 全程记录知识来源和证据等级

## 二、核心数据流

### 2.1 单次对话处理流程

```
用户输入（可带截图/语音）
  ↓
【表现层】Streamlit 接收输入，处理上传文件
  ↓
【编排层】StructPilotApp.handle()
  ↓
【智能体层】NavigatorAgent.handle_input()
  ├─ 关键词路由（concept / param / fault / progress）
  ├─ 状态机流转（pending → in_progress → passed/failed）
  └─ 产出「规则层结论」（权威，不可改）
  ↓
【编排层】_polish_reply()
  ├─ ① RAG 检索：top-k 知识片段（余弦相似度 ≥ 0.30）
  ├─ ② 图像处理：压缩到长边 ≤ 1024px
  └─ ③ LLMAgent.rewrite()
       ├─ 规则层结论（权威事实）
       ├─ RAG 检索结果（可引用参考）
       └─ 用户原始输入（含图像）
  ↓
【智能体层】MemoryAgent.capture_state()
  └─ 整个 PipelineState 序列化到 SQLite
  ↓
【表现层】UI 渲染 + 自动滚动
```

### 2.2 状态机流转

**检查点状态机**

```
pending（待开始）
  ↓ 用户："开始 cp_02"
in_progress（进行中）
  ↓ 
  ├─ 用户："完成 cp_02" → passed（通过）
  ├─ 用户："cp_02 报错" → failed（失败）
  └─ 用户："跳过 cp_02" → skipped（跳过）
```

**会话级状态追踪**

```python
PipelineState:
  - completed: List[str]      # 已完成的检查点
  - failed: List[str]         # 失败的检查点
  - skipped: List[str]        # 跳过的检查点
  - current_cp_id: str        # 当前检查点
  - checkpoint_records: Dict  # 详细记录
```

## 三、关键模块详解

### 3.1 NavigatorAgent（状态机与路由）

**职责**
- 关键词匹配与意图识别
- 检查点状态流转
- 规则层决策（参数建议/故障诊断/操作指导）

**路由优先级**（从高到低）

```python
1. concept_intro（概念解释）
   触发词："是什么", "定义", "含义", "原理", "作用"
   行为：查询术语库 → 返回概念卡

2. expert_triggers（专家建议）
   触发词："参数", "怎么设", "多少合适", "推荐"
   行为：查询参数规则 → 返回建议值和边界

3. fault_triggers（故障诊断）
   触发词："报错", "失败", "错误", "为什么"
   行为：匹配故障库 → 返回排查路径

4. progress_keywords（进度汇报）
   触发词："开始", "完成", "跳过", "做完了"
   行为：更新检查点状态 → 返回确认和下一步

5. 通用问答（其他）
   行为：走 SmartQA 管线
```

### 3.2 SmartQAEngine（问题理解与推理）

**四步管线**

```
Step 1: Understand（问题理解）
  - 识别问题类型（参数/质控/故障/概念/操作）
  - 提取软件名称、检查点ID、参数名
  - 判断是否需要图像证据

Step 2: Expand（查询扩展）
  - 关键词提取（中英文）
  - 缩写扩展（EER → Electron Event Representation）
  - 同义词补充

Step 3: Reason（推理决策）
  - 基于规则库的确定性推理
  - 基于知识库的经验推理
  - 多证据融合（官方文档 > 实验室经验 > 用户反馈）

Step 4: Compose（回答组合）
  - 按回答深度组织内容
  - 绑定证据来源
  - 标注不确定性
```

### 3.3 RAG 检索器（语义检索）

**实现细节**

```python
class RAGRetrievalOptimizer:
    def retrieve(query: str, top_k: int = 5):
        # 1. 计算 query embedding
        query_vec = embedding_model.encode(query)
        
        # 2. 加载知识库 embeddings（带缓存）
        kb_vecs = load_cached_embeddings()
        
        # 3. 计算余弦相似度
        similarities = cosine_similarity(query_vec, kb_vecs)
        
        # 4. 阈值过滤（默认 0.30）
        candidates = [(idx, sim) for idx, sim in enumerate(similarities) 
                      if sim >= SIMILARITY_THRESHOLD]
        
        # 5. 排序并返回 top-k
        return sorted(candidates, key=lambda x: x[1], reverse=True)[:top_k]
```

**缓存策略**

- **语料向量**：按 `sha256(text)` 缓存到 `embeddings_cache.json`
- **query 向量**：不缓存（每次问题都不同）
- **缓存更新**：知识库修改时自动失效

**优势**
- 纯 numpy 实现，无 C++ 依赖
- Windows 友好，部署简单
- 阈值过滤避免低相关内容混入

## 五、核心技术难点与解决方案

### 5.1 多智能体重复回复问题

**问题**：早期版本中，一次输入可能产生两条助手回复

**根因**：多条路由边同时触发，各自追加 message

**解决方案**：
1. 废弃 ExpertAgent / SOPAgent 的独立追加权限
2. 所有路由最终汇聚到 MemoryAgent
3. NavigatorAgent 成为唯一权威应答者
4. RAG/Vision/LLM 改写挂载在 `_polish_reply`，不产生额外回复

### 5.2 概念问答断链

**问题**：询问"EER 是什么文件"无法命中术语库

**根因**：五层断链（见 `A_domain_concept_落地报告.md`）

**解决方案**：
1. 扩充术语库（3 条 → 17 条）
2. 修复 `_match_glossary` 为子串匹配
3. 在 Navigator 中插入 `concept_intro` 强信号路由
4. AI 模式补充 LLM 自有知识兜底
5. 缩写扩展：`acronym_map.json`

### 5.3 RAG 检索噪音

**问题**：低相关知识片段混入，误导 LLM

**解决方案**：
1. 设置相似度阈值（默认 0.30）
2. 低于阈值直接丢弃
3. 在 prompt 中明确标注"参考知识（可引用，非权威事实）"
4. 规则层结论优先级最高

### 5.4 Windows 路径与编码

**问题**：中文路径、UTF-8 编码问题

**解决方案**：
1. 所有文件读写显式指定 `encoding='utf-8'`
2. 路径处理用 `pathlib.Path`
3. 运行时目录支持环境变量配置
4. 兜底机制：多个候选目录逐一尝试

### 6.2 图像处理优化

```python
def compress_image(image_path, max_size=1024):
    """
    压缩策略：
    1. 保持宽高比
    2. 长边缩放到 max_size
    3. JPEG 质量 85
    4. 原图保留，仅发送压缩版
    """
    img = Image.open(image_path)
    w, h = img.size
    
    if max(w, h) > max_size:
        scale = max_size / max(w, h)
        new_w, new_h = int(w * scale), int(h * scale)
        img = img.resize((new_w, new_h), Image.LANCZOS)
    
    # 转为 RGB（移除 alpha 通道）
    if img.mode != 'RGB':
        img = img.convert('RGB')
    
    # 保存到临时文件
    compressed_path = f"{runtime_dir}/compressed_{hash}.jpg"
    img.save(compressed_path, 'JPEG', quality=85)
    
    return compressed_path
```
