# StructPilot v5.1 交付与使用教程

## 交付时提交什么

推荐提交：

- `dist/StructPilot_v5.1.zip`
- `dist/StructPilot_v5.1.zip.sha256`
- `DELIVERY_AND_USER_GUIDE_V5.1.md`
- `CHANGELOG_V5.1.md`
- 可选：不含隐私数据的示例截图或演示数据

不要提交：

- `.env`、API Key、访问令牌
- `runtime/`、上传图片、上传音频、用户会话、SQLite 数据库
- `.venv/`、`__pycache__/`、`.pytest_cache/`
- `*.log`、缓存文件、临时文件
- 私有科研数据、未脱敏结果图、客户或课题组内部文件

## 接收方安装

1. 解压 `StructPilot_v5.1.zip`。
2. 确认电脑已安装 Python 3.10 或更高版本。
3. Windows 双击 `start.bat`。
4. 浏览器打开 `http://127.0.0.1:8501/`。

手动启动：

```bat
.venv\Scripts\python.exe -m streamlit run main.py --server.port 8501
```

首次启动会自动创建虚拟环境并安装依赖。如果单位网络无法联网，需要提前准备离线 wheel 包或在可联网机器上完成依赖安装后再交付环境。

## 基础模式和 AI 增强模式

基础模式不需要 API Key。它使用本地 SOP、规则、知识库、截图兜底和关键词检索，适合教学演示、离线检查和固定流程陪跑。

AI 增强模式需要在 `.env` 或设置页中配置 LLM、Embedding、视觉或语音 API。启用后可获得更自然的回答、更好的语义检索、图像理解、OCR 辅助和语音转写。

## 基本使用流程

1. 在侧边栏选择软件体系：`cryoSPARC` 或 `RELION`。
2. 点击“开始”或选择当前检查点，进入 12 步流程。
3. 选择回答深度：`简洁`、`教学`、`专家`。
4. 输入问题，或点击预设问题。
5. 上传截图或语音时，等待“上传完成、识别中、检索中、生成回答中”等状态完成。
6. 查看回答正文和来源；需要排查链路时打开“诊断信息”。
7. 完成、报错或跳过当前步骤后，继续下一检查点。
8. 在报告区导出流程记录。

## 三种回答深度

简洁：给结论、下一步、一个关键风险和证据来源，适合快速判断。

教学：给结论、原因、操作步骤、参数解释、常见错误和证据来源，适合带新手操作。

专家：给前提假设、参数边界、QC 标准、替代方案、回退路径、证据等级和不确定性，适合科研审核和复杂决策。

## 多模态输入

图片：适合上传 CTF、2D class、3D refinement、FSC、取向分布等截图。系统会显示原图缩略图、识别摘要和置信度。关键数值必须来自可见内容、OCR、用户文字或知识库，不能凭空猜测。

语音：适合口述“box size 怎么设”“这张 2D class 是否保留”等问题。语音会先转文字，再进入同一套问答链路。同一音频 hash 会复用缓存，减少重复转写。

文字：适合参数、流程、故障、概念和决策问题。建议写清软件、当前步骤、关键参数和异常现象。

## 知识库管理

在知识审核面板中可以查看和维护：

- 草稿
- 正式
- 废弃
- 来源等级
- 证据等级
- 风险等级
- 最近命中次数

正式知识会优先用于用户回答；草稿和废弃内容不应作为高风险建议的主要证据。

## 交付前检查

在交付前运行：

```bat
.venv\Scripts\python.exe verify_v5.py
.venv\Scripts\python.exe healthcheck.py
.venv\Scripts\python.exe -m pytest -q
.\build_release.ps1
```

生成 zip 后检查压缩包里没有 `.env`、`runtime/`、`.venv/`、日志、SQLite 数据库和用户上传文件。

## 常见问题

端口被占用：换端口启动。

```bat
.venv\Scripts\python.exe -m streamlit run main.py --server.port 8517
```

LLM 不可用：先用基础模式运行，再检查 API Key、Base URL、模型名和网络代理。

截图缺失：v5.1 已内置 cp_01 到 cp_12 兜底截图；外部截图目录缺失不会阻断启动。

回答不确定：科研场景下这是正常保护机制。补充截图、日志、软件、当前步骤、pixel size、粒径、目标分辨率等证据后再追问。
